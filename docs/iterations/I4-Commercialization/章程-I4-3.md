# 迭代章程 · I4-3：文档增强 - 解答用户核心疑问

> **项目经理**：针对用户反馈"结果准确性无法证明、建模思路过程不透明、模型准确性不说明"三大疑问，从用户文档、PDF 报告、帮助面板三个层级补充文档说明，提升产品可信任度与可理解性。
> **依赖**：I4-2 缺陷修复与版本对齐已完成。
> **迭代 ID**：I4-3（属于 I4 商用化交付）。

---

## 1. 目标

| 现状 | 目标 |
|------|------|
| 用户对"结果准确性如何证明"有疑问 | 在用户指南和报告中明确说明验证方法、指标含义、可复现性保证 |
| 建模思路和调优过程不透明 | 文档化完整思路流程，报告中记录关键决策 |
| 模型准确性指标缺少解释 | 在报告和帮助中说明每个指标的含义、好坏阈值 |
| 技术细节分散 | 集中在 `docs/guides/` 新增专门文档，三渠道覆盖解答 |

**范围冻结（本期必须做）**：

### 1.1 用户文档层（`docs/guides/`）
- 新增 `docs/guides/report-interpretation.md`，系统性回答三个问题：
  1. **结果准确性分析的报告，怎样证明？** → 说明验证方法论（独立测试集、K-Fold 交叉验证、Bootstrap 置信区间、统计显著性）、可复现性保证（固定随机种子、参数全记录）
  2. **建模的思路和过程，如何调优的？** → 梳理标准建模流程（数据→特征→训练→评估→调优）、每个阶段要解决的问题、XGBoost Studio 如何辅助用户
  3. **这个模型的准确性是多少，如何分析的？** → 分类/回归各指标含义、评级标准（好/中/差）、如何看置信区间、如何判断过拟合

### 1.2 报告层（PDF 自动生成）
- 在现有报告各章节增加**「解读说明」**小节：
  - 数据摘要章节：数据质量评分含义、如何解读缺失/异常分析
  - 模型评估指标章节：每个指标的简短解释、好坏参考标准
  - 训练过程章节：训练曲线解读、早停说明、过拟合诊断逻辑
  - 特征重要性 / SHAP 章节：不同重要性度量的区别、如何解读
- 保持现有可选章节机制，解读说明随章节一并可选

### 1.3 产品内帮助层（前端帮助面板）
- 在**模型评估页面**帮助面板补充各指标解释
- 在**模型训练页面**帮助面板补充训练过程诊断说明
- 在**报告页面**帮助面板提示用户可参考用户指南深入阅读

### 1.4 文档索引维护
- 更新 `docs/README.md` 新增文档索引
- 检查并修复可能的断链
- 对齐 `docs/CONVENTIONS.md` 规范

**非目标（本迭代不做）**：
- 重构整体文档架构 → 保持现有结构增量补充即可
- 多语言翻译 → 本次仅中文
- AI 自动生成解读内容扩写 → 已有叙事功能不变
- 修改核心算法逻辑 → 纯文档/文案补充，不改动功能

---

## 2. 设计约束

1. **不改变功能逻辑**：仅新增说明文字，现有功能行为不变
2. **向后兼容**：已生成的报告链接不受影响，新生成报告自动包含新增说明
3. **默认包含**：解读说明默认勾选，用户可取消勾选排除
4. **简短清晰**：报告内解读控制在每节 3-5 行，不冗长
