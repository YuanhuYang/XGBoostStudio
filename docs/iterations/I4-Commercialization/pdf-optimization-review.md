# PDF报告排版优化 · 专家评审结果

> **迭代**：I4-3-PDF-Baseline · 复杂场景审视
> **日期**：2026-04-05
> **目标**：五领域独立评审，产出分级优化清单
> **用户决定**：P0+P1 全部一次性在当前迭代实现

## 汇总统计

| 优先级 | 数量 | 说明 |
|--------|------|------|
| **P0** | 11 | 必须改，影响可读性和专业性 |
| **P1** | 14 | 应该改，提升专业感和稳定性 |
| **P2** | 10 | 可延后，不阻塞本次交付 |

---

## P0 必须改（11项）

| ID | 领域 | 问题描述 | 位置 |
|----|------|----------|------|
| P0-01 | 产品体验 | 正文字号 10px 偏小 → 提升到 11px，提升可读性 | `ST_DEFAULT["body"]` |
| P0-02 | 产品体验 | 表格内字体过小（7-8px）→ 提升到 9px，保证打印阅读 | 变量目录/SHAP表格等多处 |
| P0-03 | 产品体验 | 段间留白不足，视觉呼吸感差 → 增加各层级 `spaceBefore/spaceAfter` | 样式定义区 |
| P0-04 | 数据可复现 | 变量目录表格字体 8pt → 提升到 9pt | `_data_relations_flowables` |
| P0-05 | 数据可复现 | "与目标的关系"表格 7pt → 8pt | `_data_relations_flowables` |
| P0-06 | 数据可复现 | 高缺失率特征缺少视觉警示 → 增加浅黄色背景标注 | 变量目录表格 |
| P0-07 | 建模评估 | 模型参数KV表格列宽不合理 → 改为三列布局（参数名\|值\|说明）| `model_params` 章节 |
| P0-08 | 建模评估 | 过拟合诊断信息不够醒目 → 增加背景色块区分风险等级 | `overfitting` 章节 |
| P0-09 | 建模评估 | 评估指标表格说明列宽度不足 → 调整列宽分配 | `_metrics_table` |
| P0-10 | 系统契约 | 模块级字体注册只执行一次，失败后无法重试 → 延迟注册到每次生成报告时 | 模块导入 → 生成函数内 |
| P0-11 | 系统契约 | `ST_DEFAULT` 和 `ST_APA` 全量重复定义 → 基础样式继承+增量覆盖消除重复 | 样式定义区 |
| P0-12 | 质量门禁 | 多模型对比 ≥5个模型时列宽溢出 → 改进列宽计算策略，保证最小列宽 | `generate_comparison_report` |

## P1 应该改（14项）

| ID | 领域 | 问题描述 | 位置 |
|----|------|----------|------|
| P1-01 | 数据可复现 | 多个箱线图垂直间距不足 0.2cm → 增加到 0.5cm | `_data_relations_flowables` |
| P1-02 | 数据可复现 | 热力图缺少色标取值范围说明 → 补充文字说明 | 热力图 caption 补充 |
| P1-03 | 建模评估 | SHAP表格与图表间距不足 → 增加到 0.5cm，表格放图表下方 | `shap` 章节 |
| P1-04 | 建模评估 | 多图表连续排列缺少统一留白 → 每张图保留下方至少 0.4cm 留白 | 评估章节多张图 |
| P1-05 | 产品体验 | 页眉色块过重抢正文焦点 → 简化页眉设计，减少视觉权重 | `_header_footer` |
| P1-06 | 产品体验 | 二级标题手动编号 → 统一抽象编号逻辑保持层级一致 | 多处 `_h1_pair` 后手动写 2.1 |
| P1-07 | 产品体验 | 图表标题与图片间距偏紧 → 增加图标题下方留白 | `ST["caption"]` |
| P1-08 | 产品体验 | 执行摘要关键指标未加粗突出 → 加粗核心数字提升扫读效率 | `_executive_summary` |
| P1-09 | 系统契约 | Linux 字体路径可能不一致 → 增加更多备选搜索路径 | `_register_cn_font` |
| P1-10 | 系统契约 | 图片 `BytesIO` 未显式关闭 → 使用 `with` 语句保证释放 | `_img` 函数 |
| P1-11 | 系统契约 | APA 格式没有调整页边距 → APA 需要更宽边距，支持配置 | 文档构建参数 |
| P1-12 | 系统契约 | 大表格可能被不合理分页 → 验证 ReportLab 处理并确保合理断页 | 全章节测试 |
| P1-13 | 质量门禁 | 全章节勾选大数据页过多内存风险 → E2E 验证不 OOM | 全章节测试 |
| P1-14 | 质量门禁 | 空数据集排版降级验证 → 确认优雅降级不溢出页面 | `data_overview` |

## P2 可延后（10项）

| ID | 领域 | 问题描述 |
|----|------|----------|
| P2-01 | 数据可复现 | 随机种子分层等可复现信息缺失 → 数据概览增加展示 |
| P2-02 | 数据可复现 | 缺失值单元格缺少浅灰背景 → 增加视觉区分 |
| P2-03 | 建模评估 | 关键指标缺乏视觉高亮 → 主指标加粗或背景标注 |
| P2-04 | 产品体验 | 默认格式行高偏紧 → 适当增加 leading |
| P2-05 | 产品体验 | 封面顶部留白偏多 → 调整到 5cm 垂直居中 |
| P2-06 | 产品体验 | 页码位置可优化居中 → 当前右侧不影响阅读 |
| P2-07 | 系统契约 | 大表格跨页可能裁切不合理 → ReportLab 默认处理可接受 |
| P2-08 | 系统契约 | 样式可按格式分模块 → 当前规模单一文件可接受 |
| P2-09 | 质量门禁 | macOS 字体路径兼容性 → 已有多个备选，风险低 |
| P2-10 | 质量门禁 | 特殊字符XML转义 → 已覆盖三个核心，风险低 |
