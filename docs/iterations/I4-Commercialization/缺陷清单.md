# I4-1 质量基线 · 缺陷清单（分级）

> 分级定义：
> - **P0**：阻塞发版，必须修复
> - **P1**：影响体验，应该修复
> - **P2**：优化改进，可以延后

---

## P0（阻塞发版）

**无**。所有核心功能测试通过，没有发现阻塞发版的缺陷。

---

## P1（应该修复，不阻塞但影响体验）

| ID | 模块 | 缺陷描述 | 备注 |
|----|------|----------|------|
| P1-01 | 依赖 | Pydantic v2 弃用警告（class-based `config`） | 多个 schema 文件需要更新到 `ConfigDict` |
| P1-02 | 依赖 | `datetime.utcnow()` 弃用警告 | 需要替换为 `datetime.now(datetime.UTC)` |
| P1-03 | 打包 | PyInstaller 构建时有字体搜索警告 | 不影响运行，但构建日志有 WARNING |

---

## P2（可以延后）

| ID | 模块 | 缺陷描述 | 备注 |
|----|------|----------|------|
| P2-01 | 评估 | Bootstrap 置信区间 AC-6-02 未实现完整验证 | P2 模块范围，延后 |
| P2-02 | 评估 | 交互项诊断 AC-6-04 未验证 | P2 模块范围，延后 |
| P2-03 | 测试 | 前端单元测试覆盖率低，只有 1 个测试文件 | 可以后续增加 |
| P2-04 | 体验 | 某些错误提示文案可以更友好 | 不影响功能，优化延后 |

---

## 结论

- **P0 缺陷：0 个** → 可以进入 I4-2 缺陷修复阶段（仅 P1 三个警告修复）
- **当前 P0 范围已满足可发版条件**，修复 P1-01/02/03 后即可进入发布准备
