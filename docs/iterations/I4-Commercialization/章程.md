## 1. 目标

| 现状 | 目标 |
|------|------|
| 功能已实现，但未全量验收 | 按验收标准文档全量执行 P0 模块验收 |

**范围冻结（本期必须做）**：
- 按 [`docs/archive/legacy-product/验收标准文档.md`](../../archive/legacy-product/验收标准文档.md) 全量执行 P0 模块验收：

  - 模块 5：训练
  - 模块 6：评估
  - 模块 9：报告
- 跑通后端 `pytest` 全量，记录失败案例
- 跑通前端 `npm run typecheck` 和 `npm run test`
- 验证 I3 新增功能（报表章节选择/模板/APA格式）验收
- 产出 `缺陷清单.md`，按 P0/P1/P2 分级
- 更新 [`docs/archive/legacy-product/功能需求与验收状态.md`](../../archive/legacy-product/功能需求与验收状态.md) 执行记录

**非目标（本迭代不做）**：
- 修复缺陷 → 下一迭代 I4-2 处理，本迭代仅记录
- P1/P2 模块（预测/模型管理等）全量验收 → 延后，不阻塞本次发版
- 新增功能 → 不做
- 大范围重构 → 不做

## 2. 设计约束

1. **仅记录不修改**：本迭代以执行为主，代码改动仅限「无法跑测」的阻塞性修复
2. **证据链完整**：每个 AC 执行后必须有记录（通过/不通过/证据位置）
3. **分级清晰**：P0 = 阻塞发版；P1 = 影响体验但不阻塞；P2 = 后续优化
