# I5-Frontend-Test-Infra 章程

## 迭代目标

按调研文档 [`../../guides/frontend-ui-automation-testing.md`](../../guides/frontend-ui-automation-testing.md) 阶段一，搭建前端UI自动化测试基础设施，让首个E2E测试在CI上可运行。

## 范围冻结（必须做）

- 安装依赖：`playwright`、`@playwright/test`、`@testing-library/react`、`@testing-library/user-event`、`@testing-library/jest-dom`、`jsdom`
- 创建 `client/playwright.config.ts` 配置：
  - 配置 Web 模式测试项目（针对 `dev:web`）
  - 配置 Electron 模式测试项目（针对编译后的应用）
- 更新 `client/vitest.config.ts` 启用 `jsdom` 环境支持组件测试
- 创建 `client/src/test-setup.ts` 全局配置 Testing Library
- 更新 `.github/workflows/ci.yml` 添加 Playwright 测试作业
- 编写第一个 E2E smoke test：数据导入页面基本流程测试
- 创建 `client/e2e/` 目录结构和 `fixtures/` 放测试数据
- 添加测试 npm scripts 到 `client/package.json`

## 非目标（不做）

- **不**大规模编写P0全量测试用例（下一迭代 I5-Frontend-Test-P0-Coverage 做）
- **不**修改业务组件代码（除非解决测试阻塞的最小改动）
- **不**替换现有单元测试体系

## DoD（完成定义）

- [ ] `npm install` 成功，依赖安装无冲突
- [ ] Playwright 配置正确，支持 Web 和 Electron 双项目
- [ ] Vitest 启用 jsdom，Testing Library 可正常工作
- [ ] 首个 E2E smoke test 本地可运行通过
- [ ] GitHub Actions CI 成功运行 Playwright 测试
- [ ] 无P0缺陷（阻塞性问题）

## 目标产品版本

纳入 `0.4.0` 版本。

## 依赖

- I5-Frontend-Test-Research（调研文档完成）
- 后端服务可正常启动（测试需要后端API）

## 测试策略

- 本迭代基础设施自身测试：首个测试必须绿色
- 回归：运行现有单元测试 + 类型检查确保不破坏既有功能

## 建议上下文边界

- 独立新开 Agent 会话执行本迭代
- 范围仅限基础设施，不扩展到全量测试用例
