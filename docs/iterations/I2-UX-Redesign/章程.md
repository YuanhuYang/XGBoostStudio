# 迭代章程 · I2：信息架构与导航重构

> **项目经理**：基于 JASP 竞品分析结果，重构左侧导航信息架构，增强操作关联性与流程引导；解决「模块分散、操作不连贯」痛点。  
> **依赖**：I1 交付物（竞品分析报告、UX 需求清单、优先级矩阵）。  
> **迭代 ID**：I2-UX-Redesign。

---

## 1. 目标

| 现状 | 目标 |
|------|------|
| 左侧 11 个模块平铺分散，用户无法感知操作顺序 | 按流程分组导航，增加步骤状态联动，新手能一眼看出下一步该做什么 |
| 向导模式与专家模式割裂，切换不流畅 | 实现向导→专家平滑过渡，保持全局状态共享 |

**范围冻结（本期必须做）**：
- [x] 左侧导航按「数据准备 / 模型构建 / 模型优化 / 模型管理 / 结果输出」分组
- [x] 每个菜单项显示步骤状态（✓ 已完成 / ● 进行中 / ○ 未开始）
- [x] 分组显示已完成进度（如「数据准备：2/3 ✓」）
- [x] 高亮推荐下一步
- [x] 向导最终结果页增加「进入专家模式继续调优」按钮
- [x] 专家模式顶部增加步骤进度概览条

**非目标（本迭代不做）**：
- 报表章节定制 → 延后 I3
- 项目模板功能 → 延后 I3
- 新增参数预设 → 延后 P2 或 I4
- 帮助文档优化 → 延后 P2

---

## 2. 设计约束

1. **兼容性**：必须保持现有全局状态传递（activeDatasetId / activeSplitId / activeModelId）不变
2. **可折叠**：分组支持折叠/展开，默认全部展开
3. **响应式**：侧边栏折叠（窄模式）时，分组标签正确显示
4. **不破坏**：智能向导入口仍保留在顶部，现有功能不受影响

## 4. 分组结构建议（最终由实现调整）

```
智能向导（保持独立顶部）
─────────────────────────
📊 数据准备
  ├─ 数据导入
  ├─ 特征分析
  └─ 特征工程
⚙️ 模型构建
  ├─ 参数配置
  └─ 模型训练
📈 模型优化
  ├─ 模型评估
  └─ 模型调优
📦 模型管理
  └─ 模型管理
📄 结果输出
  ├─ 分析报告
  └─ 交互预测
```

**状态规则**：
- 菜单项状态：✓ 对应模块已完成关键操作（有数据/有划分/有模型），● 当前打开，○ 未开始
- 分组 footer 显示：`X/Y ✓`（已完成步数）

## 5. 完成定义（DoD）

### 功能验收
- [ ] P0 全部需求实现
- [ ] 端到端流程走通：欢迎页 → 数据导入 → 特征分析 → 特征工程 → 参数配置 → 训练 → 评估 → 调优 → 报告 → 预测，每个步骤状态更新正确
