# 迭代章程：I6-ThreeModeUX

> **迭代 ID**：I6-ThreeModeUX
> **产品版本**：v0.4.0
> **状态**：已完成
> **日期**：2026-04-06

---

## 一、背景与动机

v0.3.0 实现了完整的后端三域重构（数据分析/模型分析/PDF报告），前端交互架构存在明显短板：

| 问题 | 描述 |
|------|------|
| 向导模式 | 6步流程功能可用，但 `workflowStep` 未同步全局 store，侧边栏无感知 |
| 学习/调优路径（v0.3） | 早期仅 SmartWorkflow 内开关；v0.4+ 已全局 ModeSwitcher + 教学组件 |
| 专家模式 | store 类型存在但 `setWorkflowMode('expert')` 从未被 UI 调用，实质缺失 |
| 侧边栏 | 三种模式下完全相同，无模式感知，无进度指示 |
| 状态保留 | ID 全局存在，但模式切换无感知逻辑，向导恢复不同步全局 |

**目标**：落地产品概览中规划的「三种体验模式」，让向导/学习/专家三种用户路径均真实可用。

---

## 二、迭代范围（已冻结）

### P0：模式切换基础设施
- appStore 扩展：`previousMode`、`modeFirstVisit`、`isTraining` + localStorage 持久化
- `ModeSwitcher.tsx`：Header 三态 Segmented 切换器
- `MainLayout.tsx`：三态侧边栏（guidedMenu / learningMenu / expertMenu）
- 模式感知自动折叠/展开侧边栏

### P0：向导模式体验完善
- `workflowStep` 全局同步
- 向导侧边栏极简化 + 步骤圆点 + Logo 进度文字
- 向导模式离开拦截确认
- Step 5 跳转专家模式按钮
- localStorage 恢复时同步全局 ID

### P0：教学体系（智能向导 + 模型调优；专家关闭）
- `LearningPanel.tsx`：算法直觉/风险条/效果箭头
- `ParamLabModal.tsx`（从 SmartWorkflow 提取，ParamConfig 可复用）
- `ParamExplainCard` 在 `showTeachingUi`（`guided` \| `learning`）时嵌入 `LearningPanel`
- `teachingUi.ts`：统一开关；**向导模式默认开启**与调优相同的教学 UI
- `ParamConfig`：Banner + 实验按钮（非 expert 时）
- `ModelTraining` 收敛解释卡（E5）
- `FeatureAnalysis` IV/KS/PSI 概念 Popover（E6）
- `ModelEval` 指标解释 Tooltip（E7）
- `ModeOnboardingModal`：首次进入引导弹窗（E8）

### P0：专家模式实现
- 完整 10 模块菜单 + 220px 宽 + 绿色状态徽章
- 专家模式接管提示条（3秒自动关闭）
- `ModelManagement` AUC/KS 差值列（F3）
- `ModelTuning` 调优轨迹最优点 Pin 标记（F4）
- Ctrl+K 全局命令面板（F6）

### P1：模式切换安全
- SSE 训练进行中切换模式 → `ModeTransitionModal` 弹窗
- `isTraining` 信号由 `ModelTraining` 写入
