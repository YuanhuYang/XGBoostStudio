# 迭代章程 · CQ-1：质量基线（全量测试与失败清单）

**迭代 ID**：**CQ-1**（质量 · 全量测试 · 商用化路线之 **第一束**）。  
**项目经理口径**：与路线图一致 — **仅主干、本期不对外发版、不 bump SemVer**；产出为 **测试基线证据 + 失败清单**，供 **CQ-2** 认领修复与 B 档交付。

**对齐**：[`.github/workflows/ci.yml`](../../../.github/workflows/ci.yml)、[`docs/product/RELEASE_CHECKLIST.md`](../../product/RELEASE_CHECKLIST.md) §1～§2。

## 1. 范围冻结（本期必做）

| 项 | 说明 |
|----|------|
| CI 等价 | `server/`：`uv sync --all-groups --frozen` + `uv run pytest -q` |
| 客户端 | `client/`：`npm ci` + `npm run test -- --run` + `npm run typecheck` |
| 产品级补充 | `server/`：`uv run pytest`（全量，勿默认收集 `acceptance_test.py`，见 `server/tests/conftest.py`） |
| 可选/按需 | `uv run python tests/check_wizard.py`；`uv run python tests/acceptance_test.py`（需 API 已启动） |

## 2. 非目标（CQ-1）

- 不开始 **CQ-1 失败清单之外** 的缺陷修复（修复归 **CQ-2**）。
- 不新增产品功能、不做大范围重构。
- **例外**：仅当某项测试 **完全无法执行**（环境/导入/明显坏配置）时，允许 **最小** 改动以解除阻塞。

## 3. 交付定义（DoD）

- [`失败清单.md`](失败清单.md)（或本目录 [`执行记录.md`](执行记录.md) 专节）含：**命令、日志摘要、分级 P0/P1、建议 owner**；清单经确认后可作为 **CQ-2** 输入。
- 若 **零失败**：清单中明确写「无阻塞项」，并附全量命令通过摘要。

## 4. 与 CQ-2 的交接物

- **冻结** [`失败清单.md`](失败清单.md) 终稿（含复现步骤）。
- CQ-2 仅处理清单内项 + B 档（[`RELEASE_CHECKLIST.md`](../../product/RELEASE_CHECKLIST.md) §6）及回归。
