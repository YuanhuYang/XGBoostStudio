# I5-Frontend-Test-Research 章程

## 迭代目标

深入调研前端UI自动化测试方案，输出贴合 XGBoost Studio 实际项目情况的技术方案文档，为后续基础设施搭建和测试用例落地提供指导，系统化提升 UI 自动化覆盖率。

## 范围冻结（必须做）

- 分析当前项目前端架构：Electron + React + TypeScript + Vite + Vitest
- 对比至少4种主流前端UI自动化测试方案
- 选型必须考虑：Electron支持、开源免费、双模式（Web/Electron）、TypeScript、CI友好
- 输出完整的实施路线图，分阶段落地
- 输出关键用户路径测点分级建议（P0/P1/P2）
- 给出目录结构建议，与现有项目融合方案
- 识别关键风险与应对措施
- 交付完整文档：`docs/guides/frontend-ui-automation-testing.md`

## 非目标（不做）

- **不**修改任何产品代码（除文档外）
- **不**安装依赖、不配置框架（下一迭代做）
- **不**编写大量测试用例（下一迭代做）
- 不替换现有Vitest单元测试体系

## DoD（完成定义）

- [x] 文档完成，包含对比表格、选型结论、实施路线图、测点建议、风险分析
- [x] 选型结论明确，理由清晰，贴合本项目同时支持 Web 和 Electron 的实际情况
- [x] 路线图可执行，分阶段可独立交付
- [x] 与现有测试体系融合方案清晰，不破坏既有资产
