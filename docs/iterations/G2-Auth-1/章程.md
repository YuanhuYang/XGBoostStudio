## 1. 背景与目标

**问题**：专家用户质疑「同样数据、两次训练结果不一致」或「无法复现论文/内部分享时的实验」。  
**目标**：任意一次**训练**（及后续扩展：**调优**）在系统内可导出 **运行档案（Run Provenance）**，包含版本与随机性信息，使结果 **在相同代码版本与相同输入下可复现**（或 **显式记录** 不可避免的非确定性）。

## 2. 交付范围（本期 DoD）

### 2.1 必须（P0）

| # | 交付物 | 说明 |
|---|--------|------|
| 1 | **训练随机种子策略文档化** | 在代码与本文档中写明：`DatasetSplit` 的 `random_seed`、训练线程内 `numpy`/`random`/XGBoost 是否固定；若未全局固定，须列出 **未固定来源** 与 **缓解措施**（例如文档建议用户固定 `split_id` + 版本）。 |
| 2 | **`Model` 持久化 `provenance_json`（或等价）** | 至少包含：`python` 版本、`xgboost` 版本、`sklearn` 版本、`pandas` 版本（可从 `importlib.metadata` 读取）、`dataset_id`、`split_id`、训练完成时的 **`params` 最终合并结果**（与 `params_json` 一致）、`metrics_json` 摘要、**划分种子**（来自 `DatasetSplit`）。 |
| 3 | **API：`GET /api/models/{id}/provenance`**（或 `.../run-manifest`） | 返回上述结构化 JSON（200）；模型不存在 404。 |
| 4 | **pytest** | 训练后 GET provenance 字段齐全；与信任链测试可合并或独立 `test_provenance_g2_auth1.py`。 |

## 3. 验收（质量门禁 / QA）

- 冻结 AC（建议写入 [`验收-AC证据矩阵.md`](验收-AC证据矩阵.md) **G2-Auth-1** 小节）：**每条** pytest 或豁免。  
- **同机同数据同划分** 连续两次训练：`metrics` 差异在 **文档约定阈值内** 或 **标注为预期非确定**（若无法消除）。  
- CI：`uv run pytest` 全绿。

## 4. 五领域（评审要点）

| 领域 | 关注点 |
|------|--------|
| 系统与契约 | API 稳定、字段可版本演进（`schema_version` 可选） |
| 建模与评估 | 种子传递路径无遗漏 |
| 数据与可复现 | `split_id` 与叙事/训练数据同源（沿用 G1） |
| 产品体验 | 模型详情页是否展示「复制运行档案」或「导出 JSON」；帮助文档一段话解释「为何需要看版本号」 |
| 质量门禁（QA） | 快照/字段断言 + 回归清单 |

---

## 5. 风险

- **依赖升级**导致指标漂移：档案中 **必须** 记录库版本；重大升级时重跑权威集（乳腺癌/Titanic）。  
- **SQLite 与多进程**：若未来多 worker，需另议任务级锁；本期单进程假设。
