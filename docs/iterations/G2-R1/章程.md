# 迭代章程 · G2-R1：报告专业化 — 数据关系与叙事（无 LLM 优先）

> **项目经理**：组织 **五领域** 对齐目标、范围与阶段门禁（术语见 [`项目管理术语-五领域.md`](项目管理术语-五领域.md)）；**建议独立 Agent 会话**分阶段执行实现（见各阶段「建议上下文边界」）。  
> **迭代编号**：**G2-R1**（可拆为 G2-R1a 设计 / G2-R1b 实现 / G2-R1c 验收）。  
> **对标参照**：WPS AI「Excel 数据分析」类产品的 **可读结论密度** 与 **列间关系叙事**；**不默认**依赖云端大模型。

---

## 1. 背景与目标（项目经理）

| 现状 | 目标 |
|------|------|
| PDF 报告偏重 **模型侧**：指标、ROC、混淆矩阵、SHAP、参数等 | 增加 **数据侧专业叙事**：列与列关系、与业务可理解的「数据故事」、关键发现与局限 |
| 缺少 **数据源层面的关系分析** 进入报告主文 | 在 **本地计算** 前提下，输出接近「专业初稿」的 **结构化分析章节**（图表 + 规则/统计驱动的文字） |

**非目标（本章程冻结范围内）**：默认将 **原始明细** 或 **可还原明细** 的样本上传至 **公网大模型**；未部署私有化模型前，**不接**必选远程推理链路。

---

## 2. 隐私与「大模型接口」原则（系统与契约 + 项目经理）

| 策略 | 说明 |
|------|------|
| **默认** | 全流程 **本地**（现有 SQLite + 本地 Python 栈：Pandas / Scipy / Statsmodels / Sklearn）。 |
| **可选扩展（预留）** | 设置项中可增加 **「第三方 LLM API（默认关闭）」**：仅当用户 **显式开启** 且 **自行配置 Endpoint/Key** 时可用；文档须声明 **出域数据范围**（建议仅允许 **聚合统计/脱敏摘要**，禁止默认传全量原始表）。 |
| **私有化模型** | 未来若接入，走 **同一抽象接口**（provider = local_stats | openai_compatible | none），与报告生成管线解耦。 |

**结论**：本迭代 **交付价值** 建立在 **无 LLM** 路径上；LLM 仅为 **可插拔预留**，不得阻塞 G2-R1 验收。

---

## 3. 五领域讨论纪要（结构化结论）

### 数据与可复现

- 报告需新增 **「数据理解与关系」** 大节：变量类型表、缺失模式、单变量概要、**双变量关系**（数值-数值相关/散点趋势；数值-类别效应量或分布对比；类别-类别关联）。
- 与 **目标变量** 的关系必须单列子节：排序依据（互信息、ANOVA、相关等）与 **多重比较/假阳性** 的简短说明。
- 明确 **相关≠因果**，在固定话术模板中体现。

### 建模与评估

- 数据侧结论须与 **任务类型**（二分类/多分类/回归）一致；报告内交叉引用「训练所用特征子集」与「数据质量结论」。
- 若存在 **数据泄漏风险**（如时间序列误用随机划分），在数据关系章节给 **风险提示**（规则 + 元数据检测）。

### 产品体验（信息架构 + 可学性）

- **可学性**：术语旁 **一行人话**；每张关键图下 **一句读图结论**（由规则从统计量生成，而非自由生成）；避免大段公式；必要时脚注指向智能向导/模型调优下的教学卡片或帮助文档。
- **流程与版式**：报告章节顺序建议：**封面 → 执行摘要 → 数据与变量 → 关系与发现 → 建模与结果 → 局限与建议 → 附录**；UI：报告生成前可选 **「包含数据关系深度：标准 / 详细」**（影响页数与计算时间）。

### 系统与契约

- 新增 **`dataset_narrative_service`**（或扩展现有 `dataset_service` / 特征分析 API）：输入 `dataset_id` + `split_id`（或全表 profile），输出 **结构化 JSON**（图表规格 + bullet + 风险标记），再由 `report_service` 组装 PDF。
- 与现有 [`server/services/report_service.py`](../server/services/report_service.py)、[`chart_service.py`](../server/services/chart_service.py) **管道化**，避免 PDF 内硬编码巨型字符串。

### 质量门禁（QA）

- 新增/扩展 AC：**数据关系章节** 在 Titanic / Boston / Iris 上可 **回归快照**（关键统计量区间 + 段落关键词）。
- 性能：**大表**（如 DS-05）走 **采样或分层摘要** 路径须有开关与超时；验收记录中注明环境。

## 4. 技术方案方向（无大模型，实现线）

以下均为 **确定性算法 + 模板叙事**，可组合达到「WPS 式密度」的 **子集**（不必一期全做，由排期切片）。

1. **变量目录（Data Dictionary）**  
   每列：类型、缺失率、唯一值数、代表性统计量（数值：分位数；类别：Top-K 频次）。
2. **相关结构**  
   数值 Pearson/Spearman 热力图 + **高相关对** Top-N 列表与一句话解释模板。
3. **与目标的关系**  
   复用/统一现有互信息、单变量分析接口输出，生成 **排名表 + 解读句**。
4. **类别与数值交叉**  
   分组箱线图/小提琴图（已有能力则接图；无则分期）。
5. **多重共线性**  
   VIF 或相关阈值驱动的 **特征冗余警告** 段落。
6. **缺失与重复**  
   缺失矩阵示意图或汇总表 + 与目标缺失是否相关（若可算）。
7. **执行摘要**  
   由上述 JSON **规则拼装** 3～5 条 bullet（最高缺失列、最强相关对、与目标 Top 特征等）。

**与 WPS AI 的差距说明（诚实）**：无 LLM 时 **不会**产生开放式「业务故事」；通过 **丰富结构化发现 + 固定叙事模板** 提升 **专业感与说服力**，并在 UI/报告中注明「由本地统计自动生成」。

## 7. 风险与依赖

| 风险 | 缓解 |
|------|------|
| 报告体积与生成时间上升 | 标准/详细分级；大表采样；异步生成（后续迭代） |
| 统计误用（把噪声当信号） | 模板中强制 **样本量、显著性、多重比较** 提示 |
| 与 G1 全量验收并行冲突 | G2-R1 **独立分支**；合并前通过 CI；发布口径由项目经理决定是否绑定 G1 签核 |
