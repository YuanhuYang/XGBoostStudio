## P0 需求（必须完成）

### P0-1：左侧栏导航重构 · 按流程分组

**现状问题**：
- 当前左侧栏是平铺列表：智能向导 → 数据导入 → 特征分析 → 特征工程 → 参数配置 → 模型训练 → 模型评估 → 模型调优 → 模型管理 → 分析报告 → 交互预测（共 11 项平铺）
- 用户（尤其新手）无法直观感知操作顺序和关联性
- 已完成的步骤无法快速识别

**设计要求**：

1. **分组结构建议**（可调整，最终方案由实现确定）：
   ```
   📊 数据准备
     ├─ 数据导入
     ├─ 特征分析
     └─ 特征工程
   ⚙️ 模型构建
     ├─ 参数配置
     ├─ 模型训练
   📈 模型优化
     ├─ 模型评估
     ├─ 模型调优
   📦 模型管理
     └─ 模型管理
   📄 结果输出
     ├─ 分析报告
     └─ 交互预测
   ```

2. **状态联动要求**：
   - 每个分组显示分组内已完成步骤数（如「数据准备：2/3 ✓」）
   - 每个菜单项显示状态：✓ 已完成，● 进行中，○ 未开始
   - 高亮推荐的下一步（基于当前进度）
   - 点击已完成菜单项可直接跳转

3. **兼容要求**：
   - 保持智能向导仍然在顶部（推荐入口）
   - 折叠/展开分组，默认全部展开
   - 保持侧边栏可折叠功能

**验收标准**：
- [ ] 分组结构清晰，符合用户操作流程
- [ ] 状态显示正确，与实际流程一致
- [ ] 用户可通过导航快速识别下一步该做什么
- [ ] 不破坏原有功能和全局状态传递

---

### P0-2：向导模式与专家模式平滑过渡

**现状问题**：
- 用户在向导模式完成训练后，想在专家模式继续调优，需要手动点击左侧菜单
- 缺少一键切换入口
- 专家模式缺少进度概览

**设计要求**：

1. **向导模式 → 专家模式**：
   - 在向导最终结果页增加「进入专家模式继续调优」按钮
   - 点击后自动跳转到专家模式对应位置（如模型评估页面）
   - 保持全局状态不变（activeDatasetId / activeSplitId / activeModelId）

2. **专家模式增加进度概览**：
   - 在主内容区顶部增加一个步骤进度条（类似向导模式）
   - 显示当前流程走到哪一步，哪些步骤已完成
   - 点击进度条可直接跳转到对应步骤

**验收标准**：
- [ ] 一键切换功能正常，状态保持不变
- [ ] 专家模式进度概览显示正确
- [ ] 用户操作流程无断裂感

## P1 需求（建议完成）

### P1-1：报告生成增加章节选择与模板保存

**现状问题**：
- 当前报告生成默认包含所有章节，无法定制
- 不同场景需要不同内容（如汇报需要精简，存档需要完整）
- 缺少模板保存功能

**设计要求**：

1. **章节选择面板**：
   - 在生成报告前，显示可勾选的章节列表
   - 默认勾选全部，用户可取消不需要的章节
   - 章节说明：
     - 数据摘要（必选）
     - 数据关系分析
     - 参数配置
     - 训练过程
     - 模型评估指标
     - 特征重要性
     - SHAP 解释
     - 图表汇总

2. **模板保存**：
   - 用户可将当前勾选保存为「我的模板」
   - 下次生成报告可直接加载模板
   - 内置「精简汇报模板」和「完整存档模板」

3. **APA 格式借鉴**：
   - 输出格式对齐 APA 标准（图表标题、字体、间距）
   - 保持 PDF 导出兼容 Word 复制粘贴

**验收标准**：
- [ ] 章节选择功能正常，生成的报告只包含勾选章节
- [ ] 模板保存/加载功能正常
- [ ] APA 格式输出正确

---

### P1-2：项目模板与示例一键导入

**现状问题**：
- 新用户上手需要从零开始，缺少可直接运行的示例
- 用户无法保存完整工作流作为模板

**设计要求**：

1. **内置示例项目**：
   - 欢迎页增加「一键创建示例项目」选项
   - 内置 Titanic（二分类）、Boston Housing（回归）、Iris（多分类）三个示例
   - 点击后自动创建项目、导入数据、完成划分

2. **项目模板功能**：
   - 用户可将当前项目保存为模板
   - 新建项目可从模板创建（保留数据结构、特征配置、参数配置）
   - 模板不包含训练结果（每次新建需要重新训练）

**验收标准**：
- [ ] 内置示例可一键创建，运行正常
- [ ] 模板保存/创建功能正常

## P2 需求（可选，时间充裕再做）

### P2-1：更多参数预设模板

**现状问题**：
- 当前只有「快速验证」「均衡推荐」「深度训练」三个预设
- 缺少针对不同场景的预设（如防过拟合、高精度、快速实验等）

**设计要求**：
- 增加「防过拟合」预设（针对高维数据/容易过拟合场景）
- 增加「高精度搜索」预设（追求极致精度，不在乎时间）
- 增加「快速实验」预设（快速验证思路，牺牲一些精度换速度）

---

### P2-2：帮助文档内嵌优化

**现状问题**：
- 当前帮助是弹窗文字，缺少图示和步骤指引

**设计要求**：
- 每个模块帮助页增加一张流程示意图
- 增加「下一步推荐操作」链接
