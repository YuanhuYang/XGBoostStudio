## 2. 交付范围（DoD）

### 2.1 必须（P0）

| # | 交付物 | 说明 |
|---|--------|------|
| 1 | **基线方法论** | sklearn `DummyClassifier` / `DummyRegressor` **仅在训练集上 fit**，在 **测试集上** 计算指标；与 XGBoost 主模型同一 `X_test`。 |
| 2 | **`evaluation_protocol`（JSON）** | `GET /api/models/{id}/evaluation` 返回：`scheme`、`notes_zh`、`kfold_endpoint`、`time_series_split_supported`（产品能力为 **true**）、`current_split_strategy` / `current_split_is_time_ordered`；若训练期做过 K 折另含 **`cv_kfold`**。 |
| 3 | **单次 hold-out 披露** | 中文说明：单次划分 **未估计指标方差**；PDF 与模型评估页可见。 |
| 4 | **指标定义脚注** | PDF 与帮助（`MainLayout` `pageHelpMap` **model-eval**）含 Accuracy / AUC / RMSE（及任务相关项）公式或适用前提（短文）。 |
| 5 | **`POST /api/training/kfold` 契约** | JSON Body：`split_id`、`k`、`params`；OpenAPI 可发现。 |
| 6 | **pytest** | 固定数据上主模型指标不劣于 dummy（或登记 **豁免**）；`evaluation` 含 `evaluation_protocol`。 |

### 2.2 可选

| # | 交付物 | 说明 |
|---|--------|------|
| 7 | **模型评估页**：K 折结果表 | 调用 `POST /kfold` 展示 `fold_metrics` / `summary` |

### 2.3 非目标（本期不做）

- **时间序列划分**（`TimeSeriesSplit`）产品与实现：仅在 `evaluation_protocol` 中标记 **不支持**，缺口可记入 [`迭代计划.md`](迭代计划.md)。  
- AC-6-03 全量（箱线图、高亮 fold 等）。

## 3. 豁免

若数据集 **极度不平衡或极小样本** 导致「模型不优于 dummy」：**不得**强行改指标；须在 [`验收-AC证据矩阵.md`](验收-AC证据矩阵.md) 登记 **EXEMPT-G2-AUTH2-001** 与理由。
