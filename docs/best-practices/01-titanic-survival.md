# 最佳实践 1：泰坦尼克号生还预测（二分类）

> **任务类型**：二分类  
> **数据来源**：Kaggle Titanic - Machine Learning from Disaster  
> **数据集规模**：891 行 × 12 列  
> **难度**：入门

---

## 1. 任务目标

根据乘客的个人信息（票类、性别、年龄、船舱等级等）预测是否能够生还。这是机器学习领域最经典的入门练习题。

## 2. 下载数据集

你可以从以下地址下载数据集：

- Kaggle 官网：https://www.kaggle.com/c/titanic/data
- 直接下载（需要免费 Kaggle 账号）：
  - 训练集：https://storage.googleapis.com/tf-datasets/titanic/train.csv
  - 测试集：https://storage.googleapis.com/tf-datasets/titanic/eval.csv

将文件保存到本地，记住文件路径。

完整性校验（可选）：
- `train.csv` MD5: `347147a3a6822069478096338214034a`（Kaggle 官方版本）

**文件说明**：
- `train.csv`：包含完整特征和是否生还标签（891 行）
- `test.csv`：测试集（418 行），本示例我们只用 `train.csv`

## 3. 在 XGBoost Studio 中开始

### 3.1 新建项目

1. 启动 XGBoost Studio
2. 点击「新建项目」
3. 输入项目名称：`Titanic-Survival-Prediction`
4. 选择项目工作目录
5. 点击「创建」

### 3.2 导入数据

1. 在「数据导入」页面
2. 点击「上传文件」选择你下载的 `train.csv`
3. 等待上传完成

你会看到数据预览和数据概览：

- **行数**：891
- **列数**：12
- 特征包括：PassengerId, Survived, Pclass, Name, Sex, Age, SibSp, Parch, Ticket, Fare, Cabin, Embarked

**数据概览要点**：
- `Survived` 是目标列（0 = 未生还，1 = 生还）
- `Age` 有缺失值（约 177 个缺失）
- `Cabin` 缺失很多（约 687 个缺失）
- `Embarked` 有 2 个缺失

## 4. 数据预处理

XGBoost Studio 会在进入「智能预处理」后自动检测问题：

1. **缺失值处理建议**：
   - `Age`：推荐中位数填充
   - `Embarked`：推荐众数填充
   - `Cabin`：由于缺失率过高（>70%），建议删除该列

2. 点击「应用全部推荐」执行预处理。如果你对推荐策略有不同想法，也可以手动修改每个特征的处理方式。

3. 处理完成后数据质量评分会提升，点击「确认并继续」

## 5. 特征与划分

### 5.1 目标列选择

系统会自动推荐 `Survived` 作为目标列，因为它符合：
- 位于数据靠后位置
- 名称含义与标签相符
- 取值只有 0/1，符合二分类特征

确认选择即可。

### 5.2 特征选择

系统会自动排除 ID 类列（`PassengerId`、`Name`、`Ticket`），保留以下特征：

| 特征 | 类型 | 说明 |
|------|------|------|
| Pclass | 类别型 | 船舱等级（1/2/3）|
| Sex | 类别型 | 性别（male/female）|
| Age | 数值型 | 年龄 |
| SibSp | 数值型 | 同乘兄弟姐妹/配偶数量 |
| Parch | 数值型 | 同乘父母/子女数量 |
| Fare | 数值型 | 票价 |
| Embarked | 类别型 | 登船港口 |

确认选择。

### 5.3 划分数据集

- 训练/测试比例：默认 80/20 合适
- 分层采样：勾选（因为目标类别不平衡）
- 随机种子：保持默认 42 保证可复现
- 点击「划分数据集」

> 说明：不同版本的 XGBoost 或 scikit-learn 可能会导致划分结果有极其微小的差异，预期结果范围已经考虑了这种波动。

划分结果：
- 训练集：~713 行
- 测试集：~178 行

## 6. 参数配置（向导模式）

在向导模式下，选择「**⚖️ 均衡推荐**」：

系统会自动基于你的数据规模推荐参数：

| 参数 | 推荐值 | 原因 |
|------|--------|------|
| n_estimators | 100 | 样本量 < 1000，100 棵树足够 |
| max_depth | 3 | 避免过拟合，小数据集树深度不宜过大 |
| learning_rate | 0.1 | 经典默认值 |
| subsample | 0.8 | 样本采样增加鲁棒性 |
| colsample_bytree | 0.8 | 特征采样增加鲁棒性 |
| reg_lambda | 1 | 默认 L2 正则 |

确认参数后点击「继续」。

## 7. 模型训练

1. 点击「开始训练」
2. 等待训练完成（通常 < 10 秒）
3. 查看训练结果摘要：

预期结果（固定随机种子 42 下）：
- 测试集 AUC ≈ 0.85-0.88
- 测试集 Accuracy ≈ 0.78-0.82

如果你使用不同随机种子，结果可能会在小范围内波动，这属于正常现象。

这在 Titanic 任务中属于良好水平。

4. 训练完成后自动保存到模型库，点击「继续评估」

## 8. 模型评估

系统自动加载模型并生成评估报告：

### 关键指标解读

- **AUC = 0.86**：模型区分生还/未生还的能力很强（随机猜测 AUC=0.5）
- **Accuracy = 0.80**：80% 的预测正确
- **混淆矩阵**：通常对女性（特征重要性第一）预测更准确

### 特征重要性

Top 3 重要特征通常是：
1. **Sex**（性别）- 女性生还率远高于男性
2. **Pclass**（船舱等级）- 头等舱生还率更高
3. **Fare**（票价）- 票价高对应头等舱，生还率更高

### SHAP 分析

查看 SHAP 摘要图，可以直观看到：
- 女性（Sex=female）增大生还概率
- 男性（Sex=male）减小生还概率
- 更高票价（Fare）增大生还概率
- 更高船舱等级 Pclass 减小生还概率

SHAP 分析图表也会包含在最终生成的 PDF 报告中。

## 9. 生成报告

点击「生成完整报告」，等待后端生成 PDF 报告。报告包含：

1. 数据摘要与质量统计
2. 特征重要性排序
3. 训练曲线图
4. ROC 曲线 + AUC
5. 混淆矩阵
6. SHAP 特征重要性
7. 参数配置快照

**模型可追溯性**：训练完成后模型自动保存到模型库，命名格式为 `{项目名称}_{训练时间戳}`，你可以在模型管理页面找到本次训练的模型，也可添加标签 `titanic-example` 便于检索。

下载报告保存即可。

## 10. 总结与经验

### 本案例收获

1. XGBoost Studio 可以在 5 分钟内完成从 CSV 到训练报告的全流程
2. 自动预处理帮你跳过繁琐手动操作
3. AI 参数推荐对于小数据集效果已经很好
4. SHAP 解释帮助你理解哪些特征最重要，符合业务认知（性别确实是 Titanic 生还最重要因素）

### 进阶练习

1. 尝试使用「参数实验」对比不同 `max_depth`（2 vs 4）对结果的影响
2. 尝试打开「学习模式」查看每个参数的教学卡片
3. 使用贝叶斯调优进一步提升 1-2% 的 AUC
