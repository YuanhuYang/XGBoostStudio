# `docs/` 文档规范（仓库约定）

> **目的**：统一命名、目录与迭代边界，便于项目经理启动迭代时路径 **可预测**、Agent **省 Token**、协作 **不歧义**。  
> **原则**：**迭代 ID** 只出现在 **目录名** 与 **索引** 中；**需求名称、功能名** 写在各文件 **正文** 或 **章节标题**，**不**写进文件名（避免历史文件名无限增长）。

---

## 1. 顶层结构

| 路径 | 用途 |
|------|------|
| **`docs/README.md`** | 本目录索引（人类与 Agent 入口） |
| **`docs/CONVENTIONS.md`** | 本文档（规范唯一来源） |
| **`docs/product/`** | **产品 / 计划 / 验收基线**（随版本演进，文件名 **长期稳定**） |
| **`docs/guides/`** | **工程与上手**（与具体迭代 ID 无强绑定） |
| **`docs/iterations/<迭代ID>/`** | **单次迭代过程资产**：章程、设计、执行记录等（**按迭代分子目录**） |
| **`docs/evidence/`** | **跨迭代的抽样 / 专项证据**（仍避免用 **日期** 作主文件名；可用 **活动代号**，如 `抽样-F3`） |

---

## 2. 迭代 ID（目录名）

- 使用与 [`迭代计划`](product/迭代计划.md)、路线图一致的 ID，例如：`G1`、`G2-R1`、`G2-Auth-1`、`G2-Auth-2`、`G2-Auth-3`、`G2-Auth-4`、`G2-Auth-3-4`（联合验收）、`G1-quality`（质量轨专项）等。
- **新建迭代**时：由 **项目经理** 在路线图与 `iterations/` 下 **新建同名子目录**，再按 §3 放固定文件名。

---

## 3. 迭代目录内固定文件名（推荐）

每个 `docs/iterations/<迭代ID>/` 内 **优先** 只使用下列固定名称（无日期、无需求口语全称）：

| 文件名 | 含义 |
|--------|------|
| **`章程.md`** | 本迭代《迭代章程》（范围冻结、DoD、非目标） |
| **`设计.md`** | 本迭代设计说明（接口线框、数据模型等），可选 |
| **`执行记录.md`** | 本迭代验收执行记录（pytest/矩阵/签核摘要） |

- 若同一迭代 **必须** 保留第二份执行留痕：使用 **`执行记录-补充.md`**，仍 **不写日期**。
- **禁止** 在文件名中使用 **`YYYYMMDD`** 作为主区分符；历史已迁移文件在 Git 历史中可查日期。

---

## 4. `product/` 内文件名（稳定、无语义漂移）

- 使用 **固定短名**：如 `需求文档.md`、`验收标准文档.md`、`迭代计划.md`、`功能需求与验收状态.md`。  
- **不要** 把某一版需求营销名写进文件名；版本与范围以 **文档内章节** 与 **迭代计划** 为准。

---

## 5. Agent / Token 建议

- 迭代讨论 **默认 `@` 单文件**：`docs/iterations/<ID>/章程.md` 或 `docs/product/迭代计划.md`。  
- **不要** `@docs` 整目录；大材料用 **路径 + 章节名** 引用。

---

## 6. 与 Cursor 的配合

- **文档管家**：Skill **`xgboost-studio-doc-steward`**；Subagent **`/doc-steward`**（见 `.cursor/agents/doc-steward.md`）。  
- 新迭代启动时：PM 或 **`/pm`** 应给出 **新迭代目录路径** 与 §3 固定文件名清单。

---

## 7. 产品版本号与迭代 ID（关联方式）

- **产品版本号**（SemVer）是 **对外发布** 标识，见 `client/package.json` / `server/pyproject.toml`；**迭代 ID** 是 **内部工程与文档** 边界，见 §2～§3。  
- **不要** 把迭代 ID 嵌进版本号字符串；二者关系用 **[`product/版本与发布.md`](product/版本与发布.md)** 中的 **发布记录表** 维护（多对多）。  
- **项目经理** 在路线图/启动迭代时声明 **目标版本线** 或「本期不发版」；发版前后核对版本文件与上表。  
- 细则与 Skill：**`xgboost-studio-release-versioning`**。
