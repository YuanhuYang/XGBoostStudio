## 1. 目标

在 **同一数据集与划分** 下，保证下列链路 **可追溯、可自动化验证**：

`导入 → 划分 → 参数推荐（quick-config / recommend）→ 校验（validate）→ 训练合并（默认 ∪ 用户 params）→ metrics_json → 评估 API → 报告/叙事消费`

**非目标（本期）**：全仓库行覆盖率 100%；验收标准 **全部** 历史 AC；Optuna 调优 **完整 UI 流程**（见豁免）。

---

## 2. 参数合并单一事实来源（架构 / 模型训练）

训练线程内（[`training_service.training_stream`](../server/services/training_service.py)）：

1. `merged_params = { **\_default_params(task_type), **params }`（`params` 来自 `TrainRequest`，可含向导推荐值）。
2. 再按任务类型修正 `objective` / `num_class` 等（同文件内逻辑）。
3. `n_estimators` 从 `merged_params` 取出后参与逐步拟合，写回 `params_json` 时与 `merged_params` 一致。

**契约**：`POST /api/wizard/quick-config` 返回的 `params` 中 **每一个键** 经 `POST /api/params/validate` 须 `valid: true`（见 `test_trust_chain_contract.py`）。

## 3. 权威数据集清单（数据分析）

| ID | 数据集 | 来源 | 用途 |
|----|--------|------|------|
| DS-TITANIC | `tests/fixtures/titanic_train.csv` | Kaggle Titanic 训练子集（仓库固定文件） | 分类 + 数值目标 **Survived**，主链路回归 |
| DS-IRIS | `tests/fixtures/iris.csv` | Fisher Iris（仓库固定文件） | 导入/划分/列表；**字符串 species 目标** 训练易失败，本期 **不**作为训练黄金路径 |
| DS-BREAST | sklearn `load_breast_cancer` 运行时生成 CSV | [UCI / sklearn](https://scikit-learn.org/stable/modules/generated/sklearn.datasets.load_breast_cancer.html) | **权威渠道** 二分类数值特征；`test_authority_breast_cancer_pipeline` 区间断言 |

版本：以 [`server/pyproject.toml`](../server/pyproject.toml) 中 `scikit-learn`、`xgboost` 锁定版本为准；升级依赖后须重跑信任链 pytest 并必要时放宽区间。
