# XGBoostStudio 三大核心领域技术需求规格说明书

**文档定位**：本规格仅聚焦XGBoost专属数据分析、XGBoost模型结果分析、XGBoost专业PDF报告三大核心领域，明确可落地、可验证的技术实现要求，所有需求均为强制实现项，无模糊描述与非功能性内容。

---

## 一、XGBoost专属数据分析技术需求规格

### 1.1 核心定位

所有分析100%服务于XGBoost建模的有效性、合规性、可解释性，完全适配XGBoost树模型原生特性，从数据根源保障模型结果的可靠性，区别于通用统计软件的泛化EDA分析。

### 1.2 强制实现的技术需求

#### 1.2.1 建模目标对齐的标签专项分析

1. 必须实现标签口径全维度结构化记录与校验，输出标签定义、时间窗口、业务含义、正负样本划分规则，明确标签与建模任务（二分类/多分类/回归/排序）的匹配性。

2. 必须实现标签分布专项分析，输出标签统计分布、正负样本比例、时序分布稳定性、异常标签值识别结果，自动计算XGBoost训练所需`scale_pos_weight`参数基准值。

3. 必须实现标签与业务目标的一致性校验，输出标签对业务目标的覆盖度、代表性分析，明确标签是否可准确映射业务核心问题。

#### 1.2.2 XGBoost适配性特征数据分析

1. 必须实现特征基础属性分析，输出每个特征的类型、缺失率、唯一值占比、方差、分布类型；针对XGBoost原生缺失值支持特性，区分「业务含义缺失」与「数据错误缺失」，输出两类缺失的占比与处理建议。

2. 必须实现特征与标签的关联效力分析：分类任务输出每个特征的IV值、KS值、单特征AUC；回归任务输出相关系数、F检验值、单特征R²，自动筛选对目标变量有预测效力的特征，明确XGBoost建模的核心特征池。

3. 必须实现特征间共线性分析，输出特征相关系数矩阵、VIF值；针对XGBoost对共线性的鲁棒性特性，区分「致命共线性」与「可接受共线性」，输出特征筛选建议，而非无脑剔除高VIF特征。

4. 必须实现特征时序稳定性分析，输出每个特征跨时间窗口的PSI值、分布变化趋势，划分特征稳定性等级，自动剔除跨周期分布剧烈波动的不稳定特征。

5. 必须实现特征业务单调性分析，输出每个核心特征取值与标签正/负向的趋势一致性，支持用户配置业务单调性约束，自动校验不符合业务逻辑的特征趋势，为XGBoost训练的`monotone_constraints`参数提供输入依据。

#### 1.2.3 数据集划分与全链路数据泄露防控分析

1. 必须实现数据集划分规则的结构化记录与校验，明确训练集、验证集、OOT跨时间外推测试集的时间窗口、样本量、分层规则，校验划分逻辑是否符合时间序列隔离要求，杜绝随机拆分导致的泄露风险。

2. 必须实现全链路数据泄露自动化检测，覆盖三类核心泄露场景：

    - 时间穿越泄露检测：校验所有特征的生成时点是否严格早于标签生成时点，定位存在时间穿越的特征；

    - 标签泄露检测：校验特征是否包含标签的未来信息，定位与标签存在信息泄露的特征；

    - 特征工程拟合泄露检测：校验特征工程操作是否仅在训练集拟合，验证集/测试集仅做转换，定位存在数据窥探的特征处理步骤。

3. 必须输出泄露风险等级、风险位置、根因分析、修复方案，所有修复操作自动记录、可追溯。

#### 1.2.4 特征工程全流程可追溯分析

1. 必须实现特征工程全流程操作的结构化记录，包括缺失值处理、异常值处理、类别特征编码、特征衍生、特征筛选的每一步操作规则、输入输出、参数配置，保障100%可复现。

2. 必须实现每一步特征工程操作的效果分析，输出特征处理前后的分布变化、预测效力变化、稳定性变化，明确每一步操作对XGBoost建模的价值。

3. 必须实现最终入模特征池的全维度校验，输出最终入模特征的数量、类型、统计属性、效力等级、稳定性等级、单调性约束配置，明确入模特征的合规性与有效性。

## 二、XGBoost模型结果分析技术需求规格

### 2.1 核心定位

完整还原XGBoost建模与调优的全链路决策逻辑，全维度验证模型的准确性、泛化能力、可靠性，所有分析结论有数据支撑、可追溯、可复现，完整解决两大核心问题：①建模的思路和过程，如何调优的？②模型的准确性是多少，如何分析的？

### 2.2 强制实现的技术需求

#### 2.2.1 建模与调优全链路过程还原分析（解决「如何建模、如何调优」的核心问题）

1. 建模基线与目标定义分析

    - 必须输出建模任务的明确定义：任务类型、业务目标、技术评估指标、成功阈值、业务约束条件；

    - 必须输出基线模型的全维度结果，包括业务规则基线、逻辑回归基线、随机森林基线的核心指标，明确XGBoost模型的优化目标与基准线；

    - 必须输出XGBoost模型的初始配置说明，包括初始超参数的选择依据、损失函数选型依据、评估指标选型依据，明确建模初始思路。

2. 分层级超参数调优全路径可追溯分析

    - 必须实现分阶段调优过程的完整记录，严格遵循XGBoost专家级调优逻辑，分为5个固定阶段，每个阶段必须输出：调优目标、调优参数范围、优化算法、每一组试验的参数配置、对应指标结果、最优参数选择依据、本阶段调优的效果提升；

    - 5个调优阶段的固定顺序与强制要求：

        1. 迭代次数与学习率基准调优阶段：固定其他基础参数，配合早停机制，确定基础迭代轮数`n_estimators`与初始学习率`learning_rate`，输出训练/验证损失曲线、早停轮数、最优迭代次数的选择依据；

        2. 树结构复杂度调优阶段：针对`max_depth`、`min_child_weight`、`gamma`三个核心参数优化，目标是平衡模型偏差与方差，输出参数变化对应的训练/验证集指标偏差、最优树结构参数的选择依据；

        3. 采样策略调优阶段：针对`subsample`、`colsample_bytree`、`colsample_bylevel`三个采样参数优化，目标是降低模型方差、提升泛化能力，输出不同采样比例对应的OOT集指标变化、最优采样参数的选择依据；

        4. 正则化参数调优阶段：针对`reg_alpha`（L1正则）、`reg_lambda`（L2正则）两个正则化参数优化，目标是控制过拟合，输出正则化参数变化对应的过拟合风险变化、最优正则化参数的选择依据；

        5. 精细化收尾调优阶段：降低学习率、提升迭代轮数，进行最终精度优化，输出调优前后的指标对比、最终超参数的确定依据；

    - 必须输出全调优过程的参数-指标对应关系表、调优轨迹可视化图表，明确每一步参数调整带来的效果变化，完整还原调优的决策逻辑；

    - 必须输出最终超参数的完整明细表，每个参数的取值都有对应的调优过程与选择依据，无无理由的参数配置。

3. 模型训练过程全记录分析

    - 必须输出训练环境的完整信息：XGBoost版本、Python版本、依赖库版本、随机种子配置、硬件环境，保障模型100%可复现；

    - 必须输出训练过程的完整监控数据：每一轮迭代的训练集、验证集损失与核心指标变化、训练时长、早停触发条件与触发时点；

    - 必须输出过拟合防控全流程分析：训练集与验证集、OOT集的指标偏差变化、过拟合风险预警记录、对应的防控措施与效果。

#### 2.2.2 模型准确性与泛化能力全维度分析（解决「模型准确性是多少、如何分析」的核心问题）

1. 模型拟合度准确性分析

    - 必须输出训练集、验证集、OOT测试集的损失函数值、核心评估指标的对比，量化模型的拟合程度，明确判定欠拟合、合理拟合、过拟合三种状态；

    - 必须输出学习曲线分析：迭代轮数与各数据集指标的变化趋势，验证模型是否收敛、是否存在过拟合；

    - 回归任务必须输出残差分析：残差的统计分布、正态性检验、异方差检验、残差与特征的相关性分析，验证模型的拟合误差是否符合随机分布要求。

2. 模型区分度核心准确性量化分析

    - 必须按任务类型输出全维度核心准确性指标，所有指标必须同步输出训练集、验证集、OOT测试集、跨场景测试集的结果，验证模型的一致性：

        1. 二分类任务：必须输出AUC、KS、Gini系数、混淆矩阵、精准率、召回率、F1-score、LogLoss、TOP-K精准率、Lift值、Gain值，明确每个指标的计算逻辑、业务含义、数值结果；

        2. 回归任务：必须输出MAE、RMSE、R²、MAPE、MASE，明确每个指标的计算逻辑、业务含义、数值结果、误差分布分析；

        3. 多分类任务：必须输出宏平均/加权平均的精准率、召回率、F1-score、多分类混淆矩阵、Top-K准确率、LogLoss、Cohen's Kappa系数，以及每个类别的单独准确性指标；

        4. 排序任务：必须输出NDCG@K、MRR、MAP，以及每个位置的排序准确性指标；

    - 必须输出最优阈值的选择逻辑与结果：基于业务误分类成本矩阵，计算不同阈值下的精准率、召回率、业务成本/收益，明确最优业务阈值的选择依据，而非默认0.5阈值，输出最优阈值下的准确性指标；

    - 必须输出与基线模型的准确性对比：XGBoost模型与业务基线、技术基线模型的全维度指标对比，量化模型的准确性提升幅度。

3. 模型预测一致性分析

    - 必须输出预测结果的分布分析：训练集、验证集、OOT测试集的预测值分布对比、PSI值，验证模型预测结果的跨周期稳定性；

    - 必须输出分箱准确性分析：将预测概率/预测值按等频/等距分箱，输出每个分箱的实际标签均值、预测值均值、误差率，验证模型预测值与实际值的一致性，识别预测偏差较大的分箱；

    - 必须输出分客群准确性分析：按核心业务维度拆分样本，输出每个子客群的准确性指标，验证模型在不同子客群的预测准确性是否稳定，识别模型表现较差的客群。

4. 模型泛化能力与外推准确性分析

    - 必须输出OOT跨时间外推测试集的全维度准确性指标，验证模型在未来时间窗口的预测准确性，量化模型的时间衰减幅度；

    - 必须输出跨场景测试集的准确性指标，验证模型在未参与训练的业务场景/客群中的预测准确性，量化模型的泛化边界；

    - 必须输出鲁棒性压力测试结果：

        1. 特征扰动测试：对核心特征加入高斯噪声、提升缺失率，测试模型准确性指标的衰减幅度，验证模型对特征波动的抗干扰能力；

        2. 样本扰动测试：随机剔除/替换部分训练样本，重新训练模型，测试模型准确性指标的波动幅度，验证模型的稳定性；

        3. 极端场景测试：测试模型在极端值样本、黑天鹅事件样本中的预测准确性，验证模型的边界表现。

5. 坏样本根因诊断与准确性短板分析

    - 必须实现FP（假阳性）、FN（假阴性）、大误差样本的自动聚类分析，输出坏样本的核心特征共性、分布规律；

    - 必须输出坏样本预测失效的根因分析，覆盖特征覆盖度不足、特征效力不足、业务场景特殊、标签异常等维度；

    - 必须输出针对准确性短板的优化建议，明确模型的提升空间。

#### 2.2.3 模型可解释性与业务一致性分析

1. 全局可解释性分析

    - 必须输出XGBoost原生三类特征重要性：weight（分裂次数）、gain（增益贡献）、cover（样本覆盖度），按gain降序排序的核心特征排名，明确每个特征对模型预测的全局贡献度；

    - 必须输出核心特征的部分依赖图PDP、个体条件期望ICE图，明确单个特征取值对模型预测结果的边际影响趋势，验证特征影响趋势与业务逻辑的一致性；

    - 必须输出核心特征交互效应分析，明确特征间的联合影响效应，验证交互效应与业务认知的一致性。

2. 局部可解释性分析

    - 必须输出全量样本的SHAP值计算结果，包括SHAP摘要图、SHAP依赖图，明确每个特征对预测结果的影响方向与幅度；

    - 必须支持单样本的SHAP瀑布图、力导向图输出，明确单样本预测结果中每个特征的正向/负向贡献值，还原单样本预测的完整逻辑；

    - 必须输出核心样本的树分裂路径追踪结果，还原预测值的完整生成过程。

3. 业务一致性校验分析

    - 必须输出核心特征的影响趋势与业务约束规则的一致性校验结果，识别并定位不符合业务逻辑的特征影响趋势；

    - 必须输出模型预测逻辑与业务经验的匹配度分析，验证模型的准确性是符合业务逻辑的合理预测，而非基于数据巧合。

#### 2.2.4 模型风险与边界分析

1. 必须输出模型全链路风险识别结果，包括数据风险、泛化风险、业务风险、合规风险，每个风险的等级、影响范围、缓释方案；

2. 必须输出算法公平性分析结果，按指定分组字段计算不同分组的预测准确性偏差，验证模型无歧视性预测偏差；

3. 必须输出模型生命周期预测结果，基于时间衰减测试结果，明确模型的有效周期、重训阈值、迭代优化节点。

## 三、XGBoost专业PDF报告技术需求规格

### 3.1 核心定位

将前述两大领域的分析结果，以结构化、专业化、可追溯的逻辑完整呈现，100%覆盖建模过程、调优逻辑、准确性分析、可解释性分析，输出可直接用于业务汇报、技术评审、合规审计、科研发表的专业级报告，所有内容自动生成，无需用户手动编辑。

### 3.2 强制实现的技术需求

#### 3.2.1 报告核心强制原则

1. 逻辑闭环原则：报告必须遵循「业务目标→数据准备→建模调优→准确性验证→可解释性分析→结论建议」的完整逻辑链，无逻辑断点；

2. 可追溯原则：报告中的每一个结论、每一个指标、每一个参数，都必须有对应的分析过程与数据支撑，可反向追溯到原始数据与计算过程；

3. 可复现原则：报告必须包含所有保障模型可复现的信息，任何人拿到报告都可以100%复现建模过程与结果；

4. 受众适配原则：报告必须内置分层结构，同时满足业务人员、技术人员、合规审计人员的阅读需求。

#### 3.2.2 报告固定结构与强制内容要求

报告必须分为12个固定章节，每个章节的内容严格遵循以下要求，完整覆盖建模过程、调优逻辑、准确性分析：

##### 第1章：报告摘要与建模目标

强制内容：

1. 业务背景与核心痛点的明确描述；

2. 建模目标的清晰定义：业务目标、机器学习任务类型、核心评估指标、成功阈值；

3. 模型核心结论：最终模型的核心准确性指标、与基线模型的提升幅度、业务价值量化结果；

4. 报告核心内容导读，明确建模过程、调优逻辑、准确性分析的所在章节。

##### 第2章：标签与数据集专项分析

强制内容：

1. 标签口径的完整说明：标签定义、时间窗口、业务含义、正负样本划分规则；

2. 标签分布分析结果、标签与业务目标的一致性校验结果；

3. 数据源全说明：数据来源、样本周期、样本粒度、样本量、数据血缘记录；

4. 数据集划分规则：训练集、验证集、OOT测试集的时间窗口、样本量、分层规则、划分逻辑的合规性说明；

5. 全链路数据泄露检测结果、风险修复方案与最终效果。

##### 第3章：特征工程全流程分析

强制内容：

1. 原始特征池的全维度分析结果：特征数量、类型、基础统计属性、缺失率、异常值分析；

2. 特征预测效力分析结果：核心特征的IV/KS/相关系数、单特征预测能力排名；

3. 特征稳定性与单调性分析结果：特征PSI值、稳定性等级、业务单调性约束配置；

#### 3.2.3 报告内容生成强制规则

1. 所有内容必须基于前述两大领域的分析结果自动生成，无主观臆造内容，所有结论必须有数据支撑；

2. 所有专业术语必须有明确的业务含义解释，兼顾技术专业性与业务可读性；

3. 调优过程的描述必须完整还原决策逻辑，明确「为什么调这个参数」「调了之后有什么效果」「最终为什么选这个值」，而非仅罗列参数；

4. 准确性分析的描述必须明确「指标的含义是什么」「这个指标的结果说明了什么」「和基准比怎么样」「是否满足业务要求」，而非仅罗列数值；

5. 所有图表必须有明确的标题、坐标轴说明、图例、结论注释，无无意义的图表，每个图表都必须对应明确的分析结论；

6. 报告必须自动生成带跳转链接的多级目录、书签、图表编号、交叉引用，确保阅读便捷性。

#### 3.2.4 报告可视化强制规范

1. 所有可视化图表必须为XGBoost专属定制，适配树模型的分析逻辑，包括但不限于：调优轨迹图、学习曲线、ROC曲线、PR曲线、混淆矩阵热力图、Lift/Gain曲线图、特征重要性柱状图、PDP曲线图、SHAP全系列图表；

2. 所有图表必须为矢量高清格式，分辨率不低于300DPI，确保打印与放大后无模糊；

3. 图表配色必须统一规范，符合专业报告的视觉要求；

4. 所有图表必须按章节顺序编号，与正文内容一一对应，交叉引用准确。

#### 3.2.5 报告交付与定制化强制要求

1. 必须支持一键生成完整PDF报告，所有内容自动填充、图表自动渲染、目录自动生成，无需用户手动编辑；

2. 必须支持报告章节自定义：支持用户选择需要展示的章节、内容模块、图表，自定义报告的详略程度；

3. 必须支持4种预设报告模板：管理层简报版、业务执行版、技术专家版、合规审计版，每个模板自动适配对应的内容详略与侧重点；

4. 必须支持企业品牌定制：支持自定义Logo、页眉页脚、配色方案、水印、报告模板；

5. 必须支持报告的离线生成，所有生成操作完全在本地完成，无需联网。

#### 3.2.6 报告可复现性强制要求

1. 报告必须包含所有保障模型可复现的信息，包括数据划分规则、特征工程全流程、超参数明细、随机种子、训练环境；
