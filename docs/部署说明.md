# XGBoost Studio 部署说明

> 本文档分两部分：  
> **Part A** — 终端用户安装（零技术门槛）  
> **Part B** — 开发者环境搭建（贡献代码或二次开发）

---

## Part A：终端用户安装（最终用户）

### 终端用户完全不需要安装 Python、uv 或任何依赖库

XGBoost Studio 的发布包是一个 **Windows 安装程序（`.exe`）**，其中已包含：

- 完整的 Python 3.12 解释器
- 全部 Python 依赖库（xgboost、fastapi、shap、optuna 等 57 个包）
- Electron 渲染引擎
- 所有前端资源

用户安装后双击图标即可运行，**无需网络、无需安装 Python、断网全功能可用**。

### 安装步骤

1. 下载 `XGBoost Studio Setup x.x.x.exe`
2. 双击运行安装程序，按提示完成安装
3. 从桌面快捷方式启动应用

### 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / Windows 11（64位） |
| 内存 | ≥ 8 GB RAM（建议 16 GB） |
| 磁盘 | ≥ 500 MB 可用空间 |
| 网络 | 不需要（全离线运行） |
| 其他 | 无需安装 Python、Node.js、uv 等任何工具 |

### 技术原理说明

```
┌─────────────────────────────────┐
│     XGBoost Studio.exe          │  ← 用户看到并双击的程序
│                                 │
│  ┌──────────────────────────┐   │
│  │  Electron（前端渲染）     │   │  ← 内置，无需安装
│  └──────────────────────────┘   │
│  ┌──────────────────────────┐   │
│  │  xgboost-server.exe      │   │  ← PyInstaller 打包的 Python 服务
│  │  （Python 3.12 + 57个包）│   │  ← 内置，无需安装 Python
│  └──────────────────────────┘   │
└─────────────────────────────────┘
```

PyInstaller 在构建时将 Python 解释器和所有依赖库打包进 `xgboost-server.exe`，
Electron Builder 再将其与前端整合成最终安装包。用户机器上无需任何 Python 环境。

## Part B：开发者环境搭建

### 前提条件

开发者机器需要安装以下工具（一次性）：

| 工具 | 版本要求 | 下载地址 | 说明 |
|------|---------|---------|------|
| **uv** | ≥ 0.5 | https://docs.astral.sh/uv/getting-started/installation/ | Python 环境管理 |
| **Node.js** | ≥ 20 LTS | https://nodejs.org | 前端构建 |
| **Git** | 任意版本 | https://git-scm.com | 版本控制 |

> ⚠️ **不需要**手动安装 Python。uv 会自动下载并管理 Python 3.12。  
> ⚠️ **不需要** conda、virtualenv、pip 等其他 Python 工具。

---

### B.1 克隆项目并初始化

```powershell
# 1. 克隆仓库
git clone https://github.com/YuanhuYang/XGBoostStudio.git
cd XGBoostStudio

# 2. 配置国内镜像源（中国大陆网络必须，否则下载极慢）
# uv 镜像（清华大学）
$uvConfig = "$env:APPDATA\uv\uv.toml"
New-Item -ItemType Directory -Force -Path (Split-Path $uvConfig)
@"
[[index]]
url = "https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple"
default = true
"@ | Set-Content -Encoding UTF8 $uvConfig

# npm 镜像（淘宝）
npm config set registry https://registry.npmmirror.com

# 3. 安装 Python 后端依赖（uv 自动下载 Python 3.12 + 所有包）
cd server
uv sync
cd ..

# 4. 安装前端依赖
cd client
npm install
cd ..
```

### B.2 启动开发模式

```powershell
# 终端 1：启动 Python 后端服务（端口 18899）
cd server
uv run python main.py

# 终端 2：启动 Electron 前端（热更新）
cd client
npm run dev
```

验证后端正常：浏览器访问 `http://127.0.0.1:18899/health`，返回 `{"status": "ok"}` 即可。

### B.3 构建发布包

```powershell
# 一键构建（需要在项目根目录执行）
.\scripts\build-all.ps1

# 输出：dist/XGBoost Studio Setup x.x.x.exe
```

### B.4 uv 常用命令速查

| 场景 | 命令 | 说明 |
|------|------|------|
| 同步依赖 | `cd server && uv sync` | 根据 `uv.lock` 精确还原所有包 |
| 新增包 | `cd server && uv add <包名>` | 自动更新 `pyproject.toml` 和 `uv.lock` |
| 删除包 | `cd server && uv remove <包名>` | 自动更新配置 |
| 运行服务 | `cd server && uv run python main.py` | 在隔离环境中运行 |
| 运行脚本 | `cd server && uv run python <脚本.py>` | 任意 Python 脚本 |
| 查看已装包 | `cd server && uv pip list` | 列出当前环境所有包 |

### B.5 git 追踪策略（重要）

以下文件的追踪规则已在 `.gitignore` 中配置：

| 文件/目录 | 是否提交 | 原因 |
|----------|---------|------|
| `server/pyproject.toml` | ✅ **必须提交** | 依赖声明，相当于 package.json |
| `server/uv.lock` | ✅ **必须提交** | 精确版本锁，确保所有人环境一致 |
| `server/.python-version` | ✅ **必须提交** | Python 版本声明 |
| `server/.venv/` | ❌ **禁止提交** | 虚拟环境目录，体积大且平台相关 |
| `client/node_modules/` | ❌ **禁止提交** | npm 依赖目录 |
| `build/` `dist/` | ❌ **禁止提交** | 构建产物 |
| `data/` `*.db` `*.log` | ❌ **禁止提交** | 运行时用户数据 |

> **克隆后恢复环境只需两步**：  
> `cd server && uv sync`（Python 依赖）  
> `cd client && npm install`（Node 依赖）

## 常见问题

### Q：uv 安装很慢怎么办？
A：参考 B.1 配置清华大学镜像源，安装速度可提升 10 倍以上。

### Q：conda 环境可以用吗？
A：本项目不使用 conda 管理开发依赖。宿主机即使装有 conda，开发时也使用 uv 的隔离环境（`server/.venv/`），两者互不干扰。

### Q：在 CI/CD 环境如何构建？
A：CI 机器只需安装 uv 和 Node.js，然后运行 `cd server && uv sync` + `cd client && npm install`，之后执行构建脚本即可。无需配置任何 Python 版本，uv 自动处理。

### Q：终端用户安装后数据存在哪里？
A：用户数据（上传的数据集、训练好的模型、生成的报告）存储在用户的 `%APPDATA%\XGBoostStudio\` 目录，卸载应用不会删除数据。
