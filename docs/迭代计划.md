## 验收门禁与「全量通过」定义（项目经理）

1. **代码完成 ≠ 验收通过**：本节各轮「实现任务」完成仅代表开发交付。  
2. **对外材料用语（硬规则）**：**仅当** [`功能需求与验收状态.md`](功能需求与验收状态.md) **第二节** 中、**当期发布口径** 所覆盖行的「全量验收状态」**全部为「通过」**，且该文档 **第四节六角色签核** 已完成（或已批准豁免并记风险），方可在对外材料中写「**迭代计划范围已验收通过**」或等价表述；否则不得使用该措辞。须同时保留可追溯执行记录（日期、环境 A/B 档、命令输出或 AC 勾选表）。  
3. **必须执行的验证来源**：各轮正文 **「验证标准」** 小节 + [`验收标准文档.md`](验收标准文档.md) 中相关模块 AC；性能/大文件类可单独慢测 job，但须在状态表中注明证据位置。  
4. **六角色确认**：与第 2 条联动，由六角色在 [`功能需求与验收状态.md`](功能需求与验收状态.md) **签核表** 填写结论；项目经理据此更新 **下一阶段迭代** 状态。  
5. **当前事实状态（截至文档更新）**：全量 AC **尚未**全部执行完毕；已具备 **API/自动化** 与 **历史抽样（F3）** 证据——**抽样仅作过程参考，不能当作全量已通过**——详见 [`功能需求与验收状态.md`](功能需求与验收状态.md) 第二节、[`验收抽样-F3-分模块.md`](验收抽样-F3-分模块.md)。

## 现状核心缺陷（迭代启动前探查结论）

> **说明**：本节为策划阶段的 **历史探查记录**，用于对照需求来源；其中多项（如「仅 HTML 报告」「无 PDF」）已被上表「迭代一」等实现 **关闭**。当前是否仍存差距以验收执行记录与测试为准，勿仅读本节。

5. **错误信息不友好** — 多处 `HTTPException` 直接透传技术错误
6. **无商业化引导** — 无欢迎页/新手引导/帮助文档入口

- **架构师**：引入 `reportlab` 生成PDF；matplotlib生成图表后base64嵌入；不引入weasyprint（需要外部依赖）
- **测试专家**：PDF必须能被Adobe Reader/系统查看器开启，文件大小<5MB

## 架构决策记录

| 问题 | 决策 | 原因 |
|------|------|------|
| PDF库 | reportlab（纯Python） | 无外部依赖，PyInstaller打包友好；weasyprint需系统GTK |
| 图表生成 | matplotlib + Agg backend | 已间接依赖，headless环境下生成PNG字节嵌入PDF |
| 中文字体 | 依次尝试msyh.ttc/simhei.ttf/simsun.ttc，fallback Helvetica | Windows内置字体，不打包额外字体文件 |
| 软删除 | is_deleted=True而非物理删除 | 商业产品标准；保护数据完整性和关联引用 |
| K-Fold | 可选功能（高级训练选项） | 不破坏现有70/30 split的默认流程 |
| XLSX支持 | pandas.read_excel（已有openpyxl依赖） | 无需新增依赖 |
