## 一、模块 0：智能向导（SmartWorkflow）

> 以下验收用例覆盖三类用户场景：小白用户全流程（向导模式）、学生用户学习功能、专家模式全局状态传递。

---

### AC-0-01：小白用户全流程一键完成（核心场景）

**前置条件**：应用已启动，首次打开（无任何历史数据），进入「🎯 智能向导」页  
**操作步骤**：
1. Step 0：拖拽导入 `titanic_train.csv`，查看数据摘要卡片，点击「使用此数据集 →」
2. Step 1：查看 AI 推荐预处理卡片，点击「应用全部推荐」
3. Step 2：系统自动推荐 `Survived` 为目标列，保持默认 80/20 划分，点击「划分数据集 →」
4. Step 3：点击「⚖️ 均衡推荐」预设按钮，查看自动填充的参数及每个参数的一句话解释
5. Step 4：点击「开始训练」，等待训练完成，查看自然语言评价
6. Step 5：点击「生成完整报告」，等待生成，下载 HTML 报告

**预期结果**：
- [ ] 全程无需用户手动输入任何 split_id 或 model_id（状态自动传递）
- [ ] Step 0 数据摘要卡片正确显示「891 行/12 列，二分类任务，质量评分：[合理分值]」
- [ ] Step 2 目标列自动推荐 `Survived`（准确率 = 100%）
- [ ] Step 3「均衡推荐」按钮触发后，每个核心参数旁有一句数据驱动的推荐原因（非固定文本）
- [ ] Step 4 训练完成后自然语言摘要包含：AUC 数值 + 评价词（优秀/良好/待提升）+ 与随机猜测的对比
- [ ] Step 5 生成的 HTML 报告可正常在浏览器打开，包含：数据摘要、特征重要性图、混淆矩阵、ROC 曲线、SHAP 图、参数快照、评估指标（含置信区间）
- [ ] 报告文字为中文，图表清晰无乱码
- [ ] 整个流程（不含训练等待）操作步骤 ≤ 10 次点击

**失败判定**：任意步骤需用户手动输入 ID、报告生成失败、报告内容缺少任意一个上述图表类别

### AC-0-02：三预设方案按钮差异验证

**前置条件**：在向导 Step 3，已完成数据集划分（valid split_id）

1. 点击「🚀 快速验证」，记录赋值的参数

3. 点击「⚖️ 均衡推荐」，记录赋值的参数

**预期结果**：
- [ ] 快速验证：`n_estimators ≤ 50`，`learning_rate ≥ 0.2`，提示预期「30秒内出结果」
- [ ] 深度训练：`n_estimators ≥ 300`，`learning_rate ≤ 0.05`，提示预期「最高精度，训练时间较长」
- [ ] 均衡推荐：参数值介于上述两者之间，且与快速验证/深度训练的参数值明显不同
- [ ] 每个按钮附一句话预期效果描述，切换后面板内参数实时更新

### AC-0-05：学习模式 — 参数对比实验

- [ ] 系统用同一 split_id，分别训练配置 A（max_depth=3）和配置 B（max_depth=10）

- [ ] 图表下方展示 A/B 的最终验证集指标对比（Accuracy/AUC/RMSE 等）
- [ ] 配置 B（depth=10）的训练曲线应显示更快收敛但验证集指标与 A 接近或更差（典型过拟合现象），可直观感受参数差异
- [ ] 对比完成后，可选择「应用配置 A」或「应用配置 B」

import pandas as pd
import numpy as np

# DS-05: 10万行压力测试

n = 100_000

    'target': np.random.randn(n)
})
df.to_csv('large_100k.csv', index=False)

df2['label'] = np.random.randint(0, 2, n)
mask = np.random.rand(n, 10) < 0.3
df2.iloc[:, :10] = df2.iloc[:, :10].where(~mask)
df2.to_csv('missing_heavy.csv', index=False)
