# 项目管理术语：五领域签核（与 Cursor Subagent 对齐）

> **用途**：统一 **`docs/`** 与仓库 **`.cursor/skills/`** 的表述，避免「六角色 / 专家」等旧说法与当前流程不一致。  
> **生效**：自 2026-04-06 起，**§四 签核** 与过程记录中的「多视角评审」默认采用下表 **五领域**；历史文档中的「六角色」即 **旧版六列**（其中「AI 学生」与「产品设计」已合并为 **产品体验**）。

## 五领域（发布评审签核）

| 领域 | 关注点摘要 | Cursor Skill（`@` 目录名） |
|------|------------|----------------------------|
| **数据与可复现** | 上传/解析、划分与泄漏、统计口径、导入导出可追溯 | `xgboost-studio-role-data-reproducibility` |
| **建模与评估** | 训练/验证、指标、任务生命周期、模型版本与资源 | `xgboost-studio-role-modeling-eval` |
| **产品体验** | 任务流与信息架构、Electron/Web、可学性与防误、文案与错误提示 | `xgboost-studio-role-product-experience` |
| **系统与契约** | API 与错误模型、配置与打包、进程模型、可测性 | `xgboost-studio-role-systems-contract` |
| **质量门禁（QA）** | 回归面、契约缺口、E2E 最小集、环境与硬编码；**执行自动化与 DoD** 仍走测试交付 Skill | `xgboost-studio-role-quality-gate`（落地执行：`xgboost-studio-test-delivery`） |
