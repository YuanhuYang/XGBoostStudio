## 二、准确性指标详解

### 2.1 分类任务核心指标

| 指标 | 计算 | 业务含义 | 参考标准 |
|------|------|----------|---------|
| **AUC** | ROC 曲线下面积 | 模型整体判别能力，不受阈值影响 | > 0.7 可用 / > 0.85 良好 / > 0.9 优秀 |
| **KS** | max\|TPR - FPR\| | 正负样本分离程度，金融风控常用 | > 0.2 一般 / > 0.4 良好 |
| **Accuracy** | 正确数 / 总数 | 总体预测准确率（不均衡时参考意义低） | 视业务而定 |
| **F1-score** | 2 × P × R / (P+R) | 精准率与召回率的调和均值 | 不均衡场景优先 |
| **Brier Score** | 平方误差均值 | 概率预测校准质量 | 越小越好，0.25 为随机分类器 |

### 2.2 最优阈值选择

XGBoost 的 `predict()` 默认阈值为 0.5，但业务最优阈值需基于**误分类成本矩阵**确定：

```
最优阈值 = argmin{threshold} [FP_cost × FP(t) + FN_cost × FN(t)]
```

**示例（信贷风控）**：
- FP（误拒好客户）成本 = 500 元（损失利息收入）
- FN（误放坏客户）成本 = 3000 元（坏账损失）
- 最优阈值通常低于 0.5（宁愿多拒一些，避免坏账）

评估页的"阈值分析" Tab 展示了 0.1～0.9 不同阈值下的精准率/召回率/F1，可直观选择业务最优阈值。

### 2.3 回归任务指标

| 指标 | 计算 | 含义 |
|------|------|------|
| **RMSE** | √(Σ(y-ŷ)²/n) | 均方根误差，对大误差敏感 |
| **MAE** | Σ\|y-ŷ\|/n | 平均绝对误差，直观易懂 |
| **R²** | 1 - SS_res/SS_tot | 解释方差比例，0～1越大越好 |

## 三、OOT 跨时间集评估

### 什么是 OOT 评估

OOT（Out-of-Time）测试：使用**时间上更晚**的数据集测试模型，模拟模型部署后在真实时间流中的表现。

### 与普通 Hold-out 的区别

| 维度 | Hold-out 测试集 | OOT 测试集 |
|------|----------------|------------|
| 数据来源 | 与训练集同一时间段（随机/分层抽样） | 训练集之后的时间段 |
| 评估意义 | 检验拟合质量 | 检验时间泛化能力 |
| 风险 | 可能高估线上效果 | 更接近真实部署场景 |

### 时间衰减判断

```
衰减幅度 = AUC(holdout) - AUC(OOT)

< 0.01 → 稳定（可正常部署）
0.01 ～ 0.05 → 轻微衰减（建议监控后续时间点）
> 0.05 → 明显衰减（建议评估模型有效期或重训练）
```

## 四、PDP / ICE 可解释性

### PDP（偏依赖图）

**含义**：固定其他特征，只改变目标特征的取值，观察**平均预测结果**如何变化。

**用途**：
- 验证特征影响趋势是否符合业务逻辑（如：收入越高，还款概率越高）
- 与 `monotone_constraints` 对比，确认约束方向是否正确
- 发现非线性效应（U 型、倒 U 型曲线）

### ICE（个体条件期望）

**含义**：每条样本各自的依赖曲线（PDP 是 ICE 的均值）。

**用途**：
- 发现**个体异质性**——不同样本对同一特征的响应差异
- 当 ICE 线交叉时，说明存在**特征交互效应**，PDP 可能掩盖真实关系

```
实现细节：
- 网格点：20 个等间距取值（min～max）
- 样本：最多 200 个（随机采样）
- 返回：50 条 ICE 线（最多）+ PDP 均值 + 标准差
```

## 五、鲁棒性压力测试

### 5.1 特征扰动测试

对每个特征施加高斯噪声（σ = 特征标准差的 10%/50%/100%）和随机缺失（10%/30%），测试准确性衰减幅度。

```
稳定性评级：
  衰减 < 2%   → 稳定（可安全部署）
  衰减 2%～5% → 中等（建议生产监控）
  衰减 > 5%   → 高风险（模型对特征波动敏感）
```

**XGBoost 的天然优势**：由于 XGBoost 内置对缺失值的支持（学习缺失值的最优分叉方向），缺失率扰动通常影响较小。

### 5.2 样本扰动测试

随机剔除 10%/30% 的测试样本，重新评估，检验指标稳定性。指标波动 < 2% 为稳健。

### 5.3 极端值样本测试

识别 IQR 3 倍外的极端值样本，单独评估其准确性，与正常样本对比。

**意义**：
- 极端样本可能代表黑天鹅事件（金融危机、疫情等）
- 模型在极端样本上表现差可能不是问题，但需明确知道这一边界

## 六、坏样本根因诊断

### 诊断流程

```
FP/FN 样本识别
    │
    ▼
K-Means 聚类（最多 3 类）
    │
    ▼
与正确预测样本的特征均值对比
    │
    ▼
找出偏差最大的 Top-5 特征
    │
    ▼
生成根因描述和优化建议
```

### 根因典型模式

| 根因类型 | 表现 | 建议 |
|----------|------|------|
| 特征覆盖度不足 | 某类坏样本在关键特征上缺失率高 | 补充该维度数据 |
| 特征效力不足 | 坏样本与正常样本特征分布高度重叠 | 引入更强判别力的特征 |
| 业务场景特殊 | 坏样本集中在某一业务场景/地区/时段 | 针对该场景单独建模 |
| 标签异常 | FP/FN 样本集中在某一时段 | 核查标注逻辑，可能是标签噪声 |

## 七、算法公平性分析

### 什么是公平性分析

检验模型对不同人群子群（如性别、年龄段、地区）的预测结果是否存在系统性偏差。

### 核心指标

| 指标 | 计算 | 含义 |
|------|------|------|
| **DPD（人口统计公平差异）** | max(正向预测率) - min(正向预测率) | 各子群被预测为"正类"的概率差异 |
| **分组 Accuracy 差距** | max(准确率) - min(准确率) | 各子群的预测准确性差异 |
| **TPR 差距** | max(真阳率) - min(真阳率) | 各子群中真正正例被正确预测的比例差异 |

### 公平性风险等级

```
DPD > 0.1 或 Accuracy 差距 > 0.1 → 高风险（存在显著歧视性偏差）
DPD 0.05～0.1 → 中等（建议人工核查）
DPD < 0.05 → 低风险
```

**使用说明**：需提供分组字段（`group_col`），该字段须在测试集中存在。
