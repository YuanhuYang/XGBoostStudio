> **版本对应**：v0.3.0  
> **最后更新**：2026-04-06  
> **对应代码**：`server/services/feature_service.py`、`server/services/leakage_service.py`、`client/src/pages/FeatureAnalysis/`

---

## 设计理念

XGBoost Studio 的数据分析**不是通用 EDA**，而是**100% 围绕 XGBoost 建模需求设计**：

- 分析结论直接对应 XGBoost 参数（如 `scale_pos_weight`、`monotone_constraints`）
- 特征效力分析使用信息论/统计指标（IV/KS/PSI），而非泛化重要性
- 泄露检测从根源保障"离线效果好、上线不崩"
- 共线性分析区分"树模型可接受的共线性"与"致命共线性"

---

## 一、分析流程总览

```
数据导入
    │
    ▼
标签专项分析         ← 任务类型判断 / scale_pos_weight / 类别不均衡预警
    │
    ▼
特征效力排名         ← 分类: IV / KS / 单特征AUC  |  回归: Pearson r / F值 / R²
    │
    ▼
特征时序稳定性分析   ← PSI（按时间列分期对比）/ 稳定性分级
    │
    ▼
特征单调性分析       ← Spearman 趋势 → monotone_constraints 建议
    │
    ▼
共线性与 VIF 分析    ← 区分树模型可接受共线性 vs 致命共线性
    │
    ▼
全链路泄露检测       ← 标签泄露 / 时间穿越 / 拟合泄露
    │
    ▼
特征工程操作         ← 编码 / 缩放 / Box-Cox / 特征选择
    │
    ▼
数据集划分           ← 随机划分 / 时间序列划分（前60%训练/后40%测试）
```

## 二、标签专项分析

### 核心功能

输入：数据集 + 目标列名  
输出：任务类型推断 + 分布统计 + `scale_pos_weight` 推荐 + 异常标签识别

### 任务类型判断规则

| 目标列特征 | 判断结果 |
|-----------|----------|
| 唯一值 = 2 | `binary_classification`（二分类） |
| 唯一值 ≤ 20 且非浮点型 | `multiclass_classification`（多分类） |
| 其他（连续值） | `regression`（回归） |

### `scale_pos_weight` 自动计算

用于二分类不均衡样本的 XGBoost 内置参数：

```
scale_pos_weight = 负样本数 / 正样本数
```

**示例**：90 个负样本、10 个正样本 → `scale_pos_weight = 9.0`

- PSW > 10：严重不均衡，强烈建议设置（红色预警）
- PSW 3～10：轻度不均衡，建议设置（橙色提示）
- PSW < 3：基本均衡，可不设置

### 异常标签识别

对回归任务，使用 IQR 3 倍法识别极端标签值（可能是标注错误或异常事件）。

## 三、特征效力分析（IV / KS / AUC）

### 分类任务

| 指标 | 全名 | 含义 | 计算方法 |
|------|------|------|----------|
| **IV** | Information Value | 特征对目标的信息贡献量 | 等频 10 分箱后，各箱正负比率差 × WOE 求和 |
| **KS** | Kolmogorov-Smirnov | 特征对正负样本的区分能力 | 各箱累积正负比率差的最大值 |
| **单特征 AUC** | Area Under Curve | 单特征作为分类器的准确性 | `sklearn.metrics.roc_auc_score(y, feature)` |

#### IV 等级判读

| IV 值范围 | 预测效力 | 建议 |
|-----------|----------|------|
| < 0.02 | 无效 | 可剔除 |
| 0.02 ～ 0.1 | 弱 | 谨慎使用 |
| 0.1 ～ 0.3 | 中 | 有价值 |
| > 0.3 | 强 | 重要特征 |

### 回归任务

| 指标 | 含义 |
|------|------|
| **Pearson r** | 线性相关系数，r 绝对值越大效力越强 |
| **F 检验值** | 分组（等频分箱）后的组间方差 F 统计量 |
| **单特征 R²** | 单特征线性回归的决定系数 |

### 性能注意事项

- 数据集 > 50,000 行时自动采样 50,000 行（IV/KS 计算需全量特征循环）
- 等频分箱最多 10 箱，少于 10 个唯一值时自动降为唯一值数

## 四、特征时序稳定性分析（PSI）

### PSI 是什么

**Population Stability Index（群体稳定性指数）**：衡量特征在不同时间窗口的分布变化幅度。PSI 越大，说明特征在训练集和线上评分集之间的分布差异越大，模型可能已过时。

### 计算方法

1. 按时间列升序排列数据
2. 前 60% 为**基准期**（Baseline），后 40% 为**对比期**（Compare）
3. 对每个特征做等频 10 分箱（基于基准期确定 bin 边界）
4. 计算各箱占比差异：

```
PSI = Σ (C_i - B_i) × ln(C_i / B_i)
```

### PSI 等级判读

| PSI 值 | 稳定性等级 | 建议 |
|--------|-----------|------|
| < 0.1 | 稳定 | 可安全入模 |
| 0.1 ～ 0.25 | 轻微变化 | 建议关注分布变化，后续监控 |
| > 0.25 | 不稳定 | 建议剔除或重新评估，可能导致模型衰减 |

## 五、特征单调性分析（monotone_constraints 依据）

### 为什么需要单调性分析

XGBoost 支持 `monotone_constraints` 参数，可强制某个特征对预测结果的影响方向（如：信用分越高，违约概率越低，应设为 -1）。

这需要**数据支撑**：即分析特征与目标的趋势关系，判断是否存在统计意义上的单调性。

### 计算方法

1. 对特征等频 10 分箱
2. 计算每个分箱内目标变量的均值（`bin_means`）
3. 计算箱均值序列的 **Spearman 秩相关系数** ρ

### 结论判定

| ρ 绝对值 | 显著性 | 方向 | `monotone_constraints` 建议 |
|---------|--------|------|------------------------------|
| ≥ 0.9 且 p < 0.05 | 高置信度 | ρ > 0 → 递增 / ρ < 0 → 递减 | +1 / -1 |
| 0.7 ～ 0.9 且 p < 0.05 | 中等置信度 | 弱单调 | +1 / -1（中等把握） |
| < 0.4 或 p ≥ 0.05 | 无单调性 | 非单调 | 0（不加约束） |

## 六、全链路数据泄露检测

### 为什么数据泄露危害大

数据泄露会导致：
- 离线验证指标虚高（AUC 0.99），但上线后效果极差
- 模型"学会"了目标信息，而非真正学习规律
- 在监管合规场景中产生严重法律风险

### 三类泄露检测

#### 6.1 标签泄露检测

**场景**：某个特征实际上是目标列的变体（如：未来收入、事后衍生列）

**检测方法**：计算每个特征与目标列的 Pearson 相关系数和 Spearman 相关系数，超过阈值（默认 0.9）即判定为高风险。

**特殊情况**：与目标列完全相同的特征直接判定为 P0 严重风险。

```
风险等级：
  max_corr ≥ 0.99 → P0-严重（特征几乎等于目标）
  max_corr ≥ 0.95 → P0-高风险（强烈疑似泄露）
  max_corr ≥ 0.9  → P1-警告（需人工核查）
```

#### 6.2 时间穿越泄露检测

**场景**：特征的时间戳（如特征生成时点）晚于或等于标签的生成时点，即用了"未来"的数据预测"过去"的事件。

**检测方法**：逐行比对 `feature_time >= label_time`，统计泄露行比例。

**使用前提**：需要提供 `label_time_col`（标签生成时间列）和可选的 `feature_time_map`（特征→时间戳列映射）。

#### 6.3 拟合泄露检测

**场景**：特征工程（缺失值填充、标准化、目标编码等）在全数据集上 `fit`，导致验证集/测试集信息泄露到训练集。

**检测方法**：检查 Pipeline 日志中每个操作的 `fit_on` 字段：
- `train_only` → 合规
- `full_dataset` → P0 严重风险
- `unknown` → P1 警告（无法验证）

**正确做法**：所有拟合型操作（编码/缩放/填充）只在训练集调用 `fit()`，验证集/测试集只调用 `transform()`。

## 七、VIF 多重共线性分析（XGBoost 视角）

XGBoost 对共线性具有一定天然鲁棒性（树模型每步仅选最优分裂特征）。因此，评估共线性时需结合**特征效力**综合判断：

| 场景 | 建议 |
|------|------|
| VIF > 10 但 IV 都很高 | 两个特征均有价值，保留，观察模型表现 |
| VIF > 10 且一个 IV 远低于另一个 | 剔除 IV 低的特征 |
| VIF > 10 且 `monotone_constraints` 方向一致 | 考虑合并为一个综合特征 |
