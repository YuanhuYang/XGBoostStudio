> **版本对应**：v0.5.0  
> **最后更新**：2026-04-07（四种体验模式、顶栏训练划分/主模型/专家对比、数据处理侧栏、专家分析侧栏收窄）

## 一、产品定位

XGBoost Studio 是一款 **XGBoost 垂直领域的专业建模平台**，以桌面应用形式交付，**完全本地运行，无需联网**。

核心价值主张：
- 数据分析师和算法工程师无需编写代码，即可完成 XGBoost 全生命周期建模
- 内置 XGBoost 专属分析逻辑（IV/KS/PSI/单调性/泄露检测），而非通用统计分析
- 5 阶段分层调优流程，完整还原专家级调优决策逻辑，100% 可追溯
- 一键生成 12 章专业 PDF 报告，可直接用于汇报、评审、合规审计
- **四种体验模式**（智能向导 / 数据处理 / 模型调优 / 专家分析），覆盖从导入数据到交付报告的全链路，模式间可切换

### 与竞品的核心差距

| 竞品 | 核心短板 | XGBoost Studio 优势 |
|------|----------|---------------------|
| JASP 0.96 | XGBoost 非原生支持，基于 R gbm 包；无调优过程追溯 | 原生 XGBoost、5 阶段完整调优记录 |
| JMP Pro 16 | 需额外插件；无 OOT 验证、无泄露检测 | 内置全链路分析，含 OOT、泄露检测 |
| H2O.ai / DataRobot | 黑盒调参，无法还原决策逻辑；私有化成本高 | 完全本地、调优全程透明可复现 |
| 代码库（XGBoost + Sklearn） | 门槛极高，90% 工作需手动完成 | 可视化全流程，零代码输出专家级结论 |

## 二、用户画像

### 业务分析师（非技术背景用户）
- **背景**：聚焦业务决策，无需掌握算法细节，熟悉 Excel 和业务数据
- **核心诉求**：上传数据 → 快速得到可靠模型 → 一份可以直接用于汇报的专业 PDF
- **产品响应**：**智能向导模式**（`guided`，6 步全程引导），每步自动推荐配置，输出自然语言结论，一键生成专业报告。**默认开启**参数教学、预处理「深入了解」、参数实验（`showTeachingUi`）

### 分步数据准备用户
- **背景**：希望与菜单一致地完成导入、特征分析、特征工程与划分，再进入建模
- **核心诉求**：侧栏只出现数据相关页面，避免误入训练或向导
- **产品响应**：**数据处理模式**（`preprocess`）。侧栏 **数据工作台 / 特征分析 / 特征工程**；离开这三页前往其他模块需确认；**默认开启**与向导相同的教学 UI

### 建模进阶用户（固定划分上训练与调参）
- **背景**：已激活数据集，要在划分上迭代训练、超参搜索与模型管理
- **核心诉求**：侧栏聚焦「参数 → 训练 → 调优 → 管理」，入口清晰
- **产品响应**：**模型调优模式**（`learning`，界面文案「模型调优」）。进入前须已激活数据集；侧栏 **调优工作台** + **参数配置 / 模型训练 / 超参数调优 / 模型管理**（localStorage 键仍为 `learning`）

### 算法专家（评估、对比与交付）
- **背景**：已有训练好的模型，需要多维评估、注册表对比、PDF 报告与批量/交互预测
- **核心诉求**：不被训练与超参 UI 干扰，顶栏可快速切换主模型与对比集
- **产品响应**：**专家分析模式**（`expert`，界面文案「专家分析」）。侧栏 **模型工作台** + **模型评估 / 模型管理 / 分析报告 / 交互预测**（**不含**数据处理、向导、参数配置、训练、超参调优页）；顶栏在全局 **训练划分 / 主模型** 下拉之外，额外提供 **对比模型** 多选；**不展示**教学类入口

## 三、四种体验模式

四种模式共享同一套后端与数据流，全局状态 `activeDatasetId` / `activeSplitId` / `activeModelId` 在切换时完整保留。

**顶栏上下文（与历史「每模式一枚 Tag」不同）**：各模式均在 Header 展示 **训练划分**、**主模型** 下拉（Ant Design `Select`）；**专家分析** 模式额外展示 **对比模型** 多选。数据集在 **数据工作台** 或向导内选择与命名，不再单独占用顶栏 Tag。

导航与模式的权威实现见 [`02-architecture.md`](02-architecture.md)（`MainLayout.tsx` / `ModeSwitcher.tsx`）。

```
┌──────────────────────────────────────────────────────────────────────────────────────┐
│  模式切换器（Header，Segmented 四态）                                                    │
│  🎯 智能向导   🧩 数据处理   🔧 模型调优   📊 专家分析                                     │
└──────────────────────────────────────────────────────────────────────────────────────┘
     │                │                │                │
  侧栏：向导工作台    数据工作台        调优工作台        模型工作台
  仅 6 步向导        特征分析          + 参数配置         + 评估/管理
                     特征工程          训练/调优/管理      报告/预测
```

### 模式切换机制

- **切换器位置**：顶部 Header，`ModeSwitcher` 四态 Segmented
- **localStorage 持久化**：刷新或重启后恢复上次模式与侧栏折叠等
- **训练中保护**：SSE 训练进行中切换模式时弹出确认弹窗，训练不中断
- **首次引导**：每种模式首次进入时展示 `ModeOnboardingModal`
- **向导模式拦截**：在向导下点击侧栏非「向导工作台」项 → 「是否暂离向导？已保存进度」
- **数据处理模式拦截**：在数据处理三页之外的目标 → 「离开数据处理？」确认
- **进入模型调优前校验**：切换到「模型调优」时若尚未激活数据集，提示并保持当前模式
- **向导→专家桥接**：向导结果页可一键切换到专家分析以继续评估与报告

### 智能向导 6 步流程（向导模式专属）

```
Step 0: 数据准备       → 上传/选择数据集，查看质量评分（0-100），推断任务类型
Step 1: 数据分析       → 数据质量与任务类型展示、AI 预处理建议卡片（与 SmartWorkflow 实际页面对齐）
Step 2: 数据划分       → 目标列确认与训练/测试划分
Step 3: 参数配置       → 基于划分推荐 XGBoost 参数（可切换预设/高级参数）
Step 4: 一键训练       → 训练流水线（SSE）+ 评估与自然语言小结
Step 5: 结果总结       → 导出报告、历史记录、切换专家模式等
```

**Step 0 增强：全自动建模（AutoML）**  
在同一页提供可选入口：在**固定试验预算**内自动完成「目标列启发式 → 划分 → 规则推荐参数 → 多候选训练 → 按验证指标与过拟合等级惩罚排序」。产出多条 `Model` 记录，用户可用 Radio 切换主模型并跳转「结果总结」。**不承诺全局最优**，详见 [`08-automl-wizard.md`](08-automl-wizard.md)。

**命令行模式（开发者 / 脚本 / 与 UI 并行）**  
源码用户在 `server` 目录可运行 **`python -m cli.main`**（包名 `xs-studio`）：默认启动后端子进程并进入交互式 REPL，或通过 **`run <数据文件路径>`** 非交互完成上传与 AutoML；与浏览器中的前端共用同一 API 与数据库，并可用 `urls` / `run` 输出的**深链**打开前端以继续看模型对比与报告。说明见 [`08-automl-wizard.md`](08-automl-wizard.md) §2.2 与 [xs-studio CLI 指南](../guides/xs-studio-cli.md)。

### 教学增强能力（智能向导 + 数据处理 + 模型调优；专家分析关闭）

由 `showTeachingUi(workflowMode)` 统一控制（`guided`、`preprocess`、`learning` 为 true，`expert` 为 false）。

| 组件 | 位置 | 功能 |
|------|------|------|
| `LearningPanel` | `ParamExplainCard` 内 | 算法直觉 + 调大/调小效果 + 过拟合风险色阶条 |
| `ParamLabModal` | 参数配置页 / 智能向导 Step 3 | 对比两组参数配置的训练曲线和评估指标 |
| 收敛解释卡 | 模型训练页训练完成后 | 基于 train/val loss 差值自动生成收敛分析文字 |
| 概念卡片 Popover | 特征分析页 IV/KS/PSI Tab | 指标含义、判断阈值、计算原理解释 |
| 指标解释 Tooltip | 模型评估页各指标卡片 | AUC/KS/F1/R²/RMSE 等指标的自然语言解释 |
| 预处理「深入了解」 | 智能向导数据分析步 | 预处理建议卡片内可折叠展开统计原理说明 |

### 专家分析模式专属能力

| 功能 | 位置 | 说明 |
|------|------|------|
| 模型工作台 | 侧栏入口 | 专家模式默认落地页；状态说明与帮助内容与上下文联动 |
| 顶栏对比模型 | Header | 多选同划分下的其它模型，服务报告与对比阅读（规模上限与报告页一致） |
| AUC/KS 差值列 | 模型管理对比表 | 以基准模型为参照显示相对差值 |
| Ctrl+K 命令面板 | 全局 | 仅列出本模式可达页面（不含数据处理、向导、训练与超参页） |
| 接管提示条 | 进入专家分析时 | 显示已接管的划分/数据集/主模型等上下文摘要 |

**说明**：5 阶段超参调优轨迹、最优点 Pin 等仍在 **模型调优** 模式下的 **超参数调优** 页使用；专家分析模式侧栏不包含该页。

## 四、功能模块总览

UI 中页面与 `pageKey` 的对应关系见 [`02-architecture.md`](02-architecture.md)。下表按业务模块罗列能力（跨模式复用同一套页面组件）。

| 模块序号 | 模块名称 | 核心功能 |
|----------|----------|----------|
| 0 | 智能向导 | 四模式之一、6 步向导流程、历史记录恢复、**Step 0 AutoML** |
| 0b | 文档中心 | v0.5 应用内阅读 README / Wiki / guides（构建期打包，见 `docsManifest`） |
| 1 | 数据工作台 | CSV/XLSX 导入、质量评分、缺失值/异常值/重复行处理 |
| 2 | 特征分析 | IV/KS/PSI/单调性分析、泄露检测、标签专项分析、VIF、PCA |
| 3 | 特征工程与选择 | 编码/缩放/Box-Cox/选择、时间序列与随机划分 |
| 4 | 参数配置 | 参数推荐、自定义配置、教学卡片、参数实验室 |
| 5 | 模型训练 | 实时 SSE 进度、K折交叉验证、早停机制、收敛解释卡（教学模式下） |
| 6 | 模型评估 | 混淆矩阵/ROC/PR/校准、PDP/ICE、OOT验证、鲁棒性测试、坏样本诊断 |
| 7 | 超参数调优 | 5阶段分层调优、调优轨迹图（含最优点标记）、各阶段参数锁定与传递 |
| 8 | 模型管理 | 模型注册表、多维对比（含差值列）、McNemar 检验、版本管理 |
| 9 | 分析报告 | 12章 PDF 报告、4种预设模板、品牌定制 |
| 10 | 交互式预测 | 文件批量预测 + 表单手动预测、SHAP 单样本解释 |
