# XGBoost Studio 部署说明

> 本文档分两部分：  
> **Part A** — 终端用户安装（零技术门槛）  
> **Part B** — 开发者环境搭建（贡献代码或二次开发）

---

## Part A：终端用户安装

### 终端用户完全不需要安装 Python、uv 或任何依赖库

发布包由 [GitHub Releases](https://github.com/YuanhuYang/XGBoostStudio/releases) 提供，**内置** Python 3.12 运行时、全部 Python 依赖（xgboost、fastapi、shap、optuna、reportlab 等）与 Electron 前端，**无需网络、无需单独安装 Python、断网全功能可用**（安装包下载本身需联网）。

| 平台 | 推荐下载资产 | 说明 |
|------|----------------|------|
| **Windows** | `XGBoost Studio Setup x.x.x.exe`（安装版）或 portable `.exe`（免安装） | 详见 Wiki [`10-windows-distribution.md`](../wiki/10-windows-distribution.md) |
| **macOS** | `.dmg` 或 `.zip` | CI 产物**未签名/未公证**，首次打开可能需系统设置放行；见 [`11-mac-linux-distribution.md`](../wiki/11-mac-linux-distribution.md) |
| **Linux** | `.AppImage` 或 `.deb` | 与构建所用 glibc 版本相关；见 [`11-mac-linux-distribution.md`](../wiki/11-mac-linux-distribution.md) |

### 安装步骤（Windows）

1. 从 Releases 下载 `XGBoost Studio Setup x.x.x.exe`（或 portable 版）。
2. 双击运行安装程序，按提示完成安装（安装版可能需要管理员权限）。
3. 从桌面快捷方式启动应用。

**macOS / Linux**：按对应资产形态安装或运行（拖拽 dmg、`chmod +x` 后执行 AppImage、`dpkg -i` 安装 deb 等），细节见上表 Wiki 链接。

### 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | **Windows 10 / 11（64 位）**；或 **macOS / Linux（x64 或 arm64，以 Release 资产名为准）** |
| 内存 | ≥ 8 GB RAM（建议 16 GB，大数据集训练推荐） |
| 磁盘 | ≥ 1 GB 可用空间（安装包 + 运行时数据） |
| 网络 | 不需要（全离线运行） |
| 其他 | 无需安装 Python、Node.js、uv 等任何工具 |

### 用户数据存储位置

所有用户数据存储在：`%APPDATA%\XGBoostStudio\`（例：`C:\Users\你的用户名\AppData\Roaming\XGBoostStudio\`）

| 文件/目录 | 内容 |
|----------|------|
| `app.db` | SQLite 数据库（数据集记录、模型注册表、报告列表） |
| `datasets/` | 上传的原始数据文件（CSV/Excel） |
| `models/` | 训练保存的 XGBoost 模型（`.ubj` 格式） |
| `reports/` | 生成的 PDF 分析报告 |

> 卸载应用**不会**删除以上数据。需手动清除时，直接删除 `%APPDATA%\XGBoostStudio\` 文件夹。

### 技术原理

```
┌──────────────────────────────────────┐
│      XGBoost Studio Setup.exe        │  ← 下载并双击的安装包
├──────────────────────────────────────┤
│  ┌────────────────────────────────┐  │
│  │  Electron（前端渲染引擎）       │  │  ← 内置，展示应用界面
│  └────────────────────────────────┘  │
│  ┌────────────────────────────────┐  │
│  │  内置后端（Windows 为 .exe，     │  │  ← PyInstaller 打包的 Python 服务
│  │  macOS/Linux 为无扩展名二进制） │  │  ← 自动在后台启动，监听 127.0.0.1:18899
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

应用启动时，Electron 主进程自动拉起内置后端，健康检查通过后显示主界面。关闭应用时后端进程自动退出。

## Part B：开发者环境搭建

### 前置工具（一次性安装）

| 工具 | 版本要求 | 下载地址 | 说明 |
|------|---------|---------|------|
| **uv** | ≥ 0.5 | https://docs.astral.sh/uv/getting-started/installation/ | Python 环境管理，自动安装 Python 3.12 |
| **Node.js** | ≥ 20 LTS | https://nodejs.org | 前端构建 |
| **Git** | 任意 | https://git-scm.com | 版本控制 |

> ⚠️ **不需要**手动安装 Python，uv 会自动下载并隔离管理 Python 3.12  
> ⚠️ **不需要** conda、virtualenv、pip 等其他 Python 工具

---

### B.1 克隆并初始化

```powershell
# 克隆仓库
git clone https://github.com/YuanhuYang/XGBoostStudio.git
cd XGBoostStudio
```

**国内网络配置镜像源（中国大陆强烈推荐，可提速 10 倍）：**

```powershell
# uv PyPI 镜像（清华大学）
$uvConfig = "$env:APPDATA\uv\uv.toml"
New-Item -ItemType Directory -Force -Path (Split-Path $uvConfig)
@"
[[index]]
url = "https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple"
default = true
"@ | Set-Content -Encoding UTF8 $uvConfig

# npm 镜像（淘宝）
npm config set registry https://registry.npmmirror.com
```

**安装依赖：**

```powershell
# Python 后端依赖（uv 自动下载 Python 3.12 + 全部包）
cd server
uv sync
cd ..

# 前端依赖
cd client
npm install
cd ..
```

---

### B.2 启动开发模式

**开两个终端，先后端、后前端：**

```powershell
# 终端 1：启动 Python 后端（端口 18899）
cd server
uv run python main.py
# 看到 "Uvicorn running on http://127.0.0.1:18899" 即就绪
```

```powershell
# 终端 2：启动 Electron 前端（热更新）
cd client
npm run dev
# Electron 窗口自动弹出
```

验证后端正常：`http://127.0.0.1:18899/health` 返回 `{"status":"ok"}` 即可。  
Swagger 交互文档：`http://127.0.0.1:18899/docs`

### B.5 构建发布包

**Windows（PowerShell，仓库根目录）：**

```powershell
# 全量构建（推荐，先构建后端再构建前端）
.\scripts\build-all.ps1

# 仅构建后端（PyInstaller → server/dist/xgboost-server.exe）
.\scripts\build-server.ps1

# 仅构建前端（electron-builder → dist/XGBoost Studio Setup x.x.x.exe）
.\scripts\build-client.ps1
```

**macOS / Linux（Bash，仓库根目录）：**

```bash
chmod +x scripts/build-all.sh
./scripts/build-all.sh
```

> ⚠️ Windows 下构建前端时需要 `server/dist/xgboost-server.exe` 已存在（由 `build-server.ps1` 或 `build-all.ps1` 生成）。  
> ⚠️ Unix 下对应产物为 `server/dist/xgboost-server`（无扩展名），由 `build-all.sh` 同步至 `client/resources/xgboost-server`。  
> ⚠️ 构建期间 `dist\win-unpacked\XGBoost Studio.exe` 不能处于运行状态，否则 rcedit 会报"Unable to commit changes"。

### B.7 Git 追踪策略

| 文件/目录 | 是否提交 | 说明 |
|----------|---------|------|
| `server/pyproject.toml` | ✅ 必须提交 | 依赖声明（相当于 package.json） |
| `server/uv.lock` | ✅ 必须提交 | 精确版本锁，确保所有人环境一致 |
| `server/.python-version` | ✅ 必须提交 | Python 版本声明 |
| `server/.venv/` | ❌ 禁止提交 | 虚拟环境，体积大且平台相关 |
| `client/node_modules/` | ❌ 禁止提交 | npm 依赖目录 |
| `server/dist/` `client/dist/` `dist/` | ❌ 禁止提交 | 构建产物 |
| `*.db` `datasets/` `models/` `reports/` | ❌ 禁止提交 | 运行时用户数据 |

> 克隆后恢复开发环境只需两步：  
> `cd server && uv sync`  
> `cd client && npm install`

### B.8 macOS 开发者快速开始

macOS 上运行流程与 Windows 基本一致，只是使用 Terminal 而非 PowerShell。

**前置工具（Homebrew）：**

```bash
# 安装 Homebrew（若未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Node.js（含 npm）
brew install node

# 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**克隆并启动：**

```bash
git clone https://github.com/YuanhuYang/XGBoostStudio.git
cd XGBoostStudio

# 安装依赖
cd server && uv sync && cd ..
cd client && npm install && cd ..

# 终端 1：启动后端
cd server && uv run python main.py

# 终端 2：启动前端（Electron 窗口）
cd client && npm run dev
```

> **提示**：macOS 首次运行未签名 Electron 时，系统会弹出安全警告。  
> 在「系统偏好设置 → 安全性与隐私」中选择"仍要打开"即可。  
> 或者使用浏览器模式替代：`cd client && npm run dev:web`

### B.9 Linux 浏览器访问模式

Linux 环境不依赖 Electron（无显示器或无 GUI 均可），前端以纯 Web 方式运行。

**前置工具（Ubuntu/Debian）：**

```bash
sudo apt update && sudo apt install -y nodejs npm git curl

# 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc  # 或 source ~/.zshrc
```

**克隆并启动：**

```bash
git clone https://github.com/YuanhuYang/XGBoostStudio.git
cd XGBoostStudio

# 安装依赖
cd server && uv sync && cd ..
cd client && npm install && cd ..

# 方式 A：分开两个终端（推荐调试）
# 终端 1：启动后端
bash scripts/start.sh --server

# 终端 2：启动前端（Web 模式）
bash scripts/start.sh --client
# scripts/start.sh 会自动检测 Linux 并调用 npm run dev:web

# 方式 B：后端后台 + 前端前台
cd server && nohup uv run python main.py > /tmp/xgb-server.log 2>&1 &
cd ../client && npm run dev:web
```

**访问应用：**

```
http://localhost:5173    # Vite 开发服务（热更新）
http://127.0.0.1:18899/docs  # 后端 Swagger 调试文档
```

> **原理**：前端 React 应用在浏览器中运行时，会轮询 `http://127.0.0.1:18899/health`  
> 等待后端就绪后自动显示主界面，无需 Electron IPC 通信。

### Q：uv sync / npm install 很慢怎么办？
**A**：参考 B.1 配置国内镜像源（PyPI 用清华，npm 用淘宝），速度可提升 10 倍以上。

### Q：后端启动失败，提示"端口 18899 已被占用"？
**A**：执行 `netstat -ano | findstr 18899` 查找占用的进程 PID，然后 `taskkill /PID <PID> /F` 结束进程后重试。

### Q：构建时报"rcedit Unable to commit changes"？
**A**：`dist\win-unpacked\XGBoost Studio.exe` 正在运行，关闭应用后重新执行构建脚本。

### Q：conda 环境与 uv 冲突吗？
**A**：不冲突。宿主机即使有 conda，开发时仍然使用 uv 的隔离环境（`server/.venv/`），两者互不干扰。开发时只使用 uv，不使用 conda。

### Q：用户数据在哪里？卸载会丢失吗？
**A**：数据在 `%APPDATA%\XGBoostStudio\`。卸载应用**不会**删除此目录，数据安全保留。

### Q：支持 macOS / Linux 吗？
**A**：
- **macOS**：开发与 **桌面发布包（dmg/zip）** 均支持；发布包见 [Releases](https://github.com/YuanhuYang/XGBoostStudio/releases) 与 Wiki [`11-mac-linux-distribution.md`](../wiki/11-mac-linux-distribution.md)。开发模式可用 `bash scripts/start.sh --all` 或 `python scripts/dev.py --all`。
- **Linux**：支持 **桌面发布包（AppImage/deb）**（同上 Wiki）；开发调试仍可用 `bash scripts/start.sh --all` 或 `npm run dev:web` 在浏览器访问 `http://localhost:5173`。
