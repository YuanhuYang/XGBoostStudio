# 功能需求与验收状态（六轮交付对齐）

> **用途**：把 [`迭代计划.md`](迭代计划.md) 中已实现能力沉淀为可追踪矩阵，并与 [`验收标准文档.md`](验收标准文档.md) 模块/AC 对应，支撑 **全量验收** 与发布门禁。  
> **原则**：**发布前**须本表「全量验收状态」对当期发布范围全部为 **通过**（或经项目经理书面豁免并记风险）；不得以「代码已合并」替代「验收已执行」。

**对外表述门禁（硬规则）**：仅当 **第二节** 表格中、**当期发布口径** 所覆盖各行的「全量验收状态」**全部为「通过」**，且 **第四节六角色签核表** 已填写结论（「通过 / 附条件 / 不通过」及日期；或有经批准的书面豁免清单并登记风险）后，方可在 **对外材料**（官网、商务与合同附件、发版说明、对客演示材料等）中使用「**迭代计划范围已验收通过**」或语义等价的表述。否则只能描述为「功能已交付 / 开发完成 / 部分验收通过」等，并指向本表与执行记录。
