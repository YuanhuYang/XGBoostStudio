# 领域：数据与可复现（Data & Reproducibility）

## 一句话

数据从进入到结果是否 **科学、无泄漏、可核对**；导出与摘要是否 **诚实、可追溯**。

## 审视要点（按需选 3–6 条写）

- 上传/解析：格式假设、编码、大文件、缺列与类型推断。
- 划分：随机种子、分层、目标列；**泄漏**与「用错集评估」风险。
- 分布与缺失：不平衡、缺失处理在 UI/API 中是否表述清楚。
- 导出：CSV/图表/PDF 与指标能否回溯到同一 `split`/任务。

## 输出（四段，禁止空话）

1. **关注点**（3–6 条）
2. **现状判断**（`server/`、`client/` 路径；不确定写「待验证」）
3. **建议**（**P0/P1/P2** + 理由 + 位置）
4. **开放问题**

合并多角色时遵循 **`xgboost-studio-multi-agent-review`**。

## PM 何时可跳过本会话

本迭代 **不涉及** 数据管线、划分、导入导出、报表数据口径时，路线图标注 **跳过**，Token 留给实现与质量门禁。
