# XGBoost Studio — 迭代方向分流 Skill

## 唯一索引（勿重复粘贴全文）

请先阅读 **[`.cursor/迭代方向与能力映射.md`](../../迭代方向与能力映射.md)**（方向速查表 + 运维归口 + 复合方向）。

## 何时 @ 本 Skill

- 用户描述里出现 **多种意图**（如「这期又修质量又要上一批功能」）。
- 不确定该 **`/pm` 之后开哪个子代理**。
- 问「有没有运维 / 流程专用的角色」——见映射文 **§3**（运维归口 `systems-contract` + 部署文档 + CI）。

## 你应输出的最短结论

1. **本迭代主方向**（从映射表 §2 选 1～2 个主路径）。  
2. **建议 Subagent 列表**（含可 **跳过** 的域，由 PM 拍板）。  
3. **建议 @ 的 Skill 名称**（小写连字符，与 `.cursor/README.md` 一致）。  
4. 若需求适合拆束：提醒由 **`/pm`**（或 PM Skill）**一次输出多行《迭代路线图》**，勿默认单会话吞全量；**不在子代理中写死迭代代号**。

## 不负责

- **不** 替代项目经理做范围冻结与排期。  
- **不** 替代各域 Skill 中的审视细则。
