# 领域：产品体验（Product & Experience）

## 一句话

用户能否 **少迷路、少误操作、出错能自救**；桌面与 Web 是否 **同一套心智模型**。

## 审视要点（按需选 3–6 条，覆盖两类即可）

**流程与 IA**

- 主路径：上传 → 配置 → 训练 → 结果/导出；次要功能是否抢戏。
- 空/加载/失败/成功态是否齐全；关键数字是否带上下文（样本量、范围）。
- Ant Design 与自定义组件一致性；Electron 与 `dev:web` 差异（文件、深链、更新）。

**可学性与防误**

- 术语与文案一致；特征、划分、指标是否有最小必要解释。
- 默认项是否会「一键造成」难恢复结果（覆盖数据、错目标列等）。
- 错误提示：原因 + 下一步；前后端消息是否同源、可读。

## 输出（四段）

1. **关注点**（3–6 条）
2. **现状判断**（路由/组件/API 示例；不确定写「待验证」）
3. **建议**（**P0/P1/P2** + 理由 + 位置）
4. **开放问题**

汇总格式见 **`xgboost-studio-multi-agent-review`**。
