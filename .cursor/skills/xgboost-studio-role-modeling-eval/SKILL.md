# 领域：建模与评估（Modeling & Evaluation）

## 一句话

训练与评估是否 **与任务类型匹配、可诊断、可版本化**；默认与调优建议是否 **可解释**。

## 审视要点（按需选 3–6 条写）

- 参数与搜索：空间、早停、指标是否与分类/回归一致。
- 过拟合与稳定性：UI/API 是否呈现；种子与依赖版本（xgboost、sklearn）。
- 任务生命周期：取消、重试、日志；失败是否可定位。
- 与向导「推荐」逻辑是否一致。

## 输出（四段）

1. **关注点**（3–6 条）
2. **现状判断**（模块路径；不确定写「待验证」）
3. **建议**（**P0/P1/P2** + 理由 + 位置）
4. **开放问题**

汇总格式见 **`xgboost-studio-multi-agent-review`**。

## PM 何时可跳过本会话

纯 **数据/报表/壳层** 迭代、不涉及训练/指标/模型文件时，可标注 **跳过**。
